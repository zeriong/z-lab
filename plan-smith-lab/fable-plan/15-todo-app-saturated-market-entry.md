# 포화 시장(할일 관리 앱) 진입 전략
- 추론 프레임: Analogical
- 한 줄 요약: 성공 유비 2개 + 실패 유비 1개를 해부한다. 실패 사례(Arc)가 성공 사례보다 더 많은 것을 가르친다 — "더 나은 재해석"만으로는 포화 시장의 전환 비용을 못 넘는다.

## 프레임이 요구하는 시작점 — 세 유사 사례의 구조 해부

포화 시장을 뚫(으려 하)던 실제 사례를 고르되, **성공만 고르지 않는다.** 성공 유비만 모으면 생존 편향 위에 전략을 세우게 된다. 실패(부분 실패) 사례 하나를 반드시 포함한다.

**사례 A — Obsidian (포화된 노트 앱 시장, Evernote/Notion의 세계)**
- 구조: 로컬 마크다운 파일(데이터 소유권) + 플러그인 생태계(사용자가 제품을 확장) + 커뮤니티 주도 성장(마케팅비 ~0) + 개인 무료·상업/동기화 유료.
- 추출 패턴 P-A: **"데이터 소유권 + 확장 생태계"로 파워유저의 영구 거점이 되기.** 대중이 아니라, 도구에 자기 시스템을 만드는 소수가 표적. 이들은 이탈률이 극도로 낮고 전도사가 된다.

**사례 B — Superhuman (포화된 이메일 클라이언트 시장, 무료 Gmail의 세계)**
- 구조: 극단적 속도와 키보드 워크플로 + 고가($30/월) + 수동 온보딩 — "모두의 이메일"이 아니라 "이메일이 직업의 병목인 소수"에게 판다.
- 추출 패턴 P-B: **니치 프리미엄 — 고통이 가장 큰 소수에게 비싸게.** 무료 지배자와 가격으로도 기능 수로도 경쟁하지 않는다.

**사례 C — Arc 브라우저 (포화된 브라우저 시장, 실패 유비)**
- 구조: 브라우저의 전면 재해석(수직 탭, 스페이스, 미학) + 열광적 초기 팬 → 그러나 대중 확장에 실패하고 개발 중단(방향 전환). 
- 추출 반(反)패턴 P-C: **"객관적으로 더 나은 재해석"은 전환 비용을 이기지 못한다.** 브라우저·할일 앱처럼 습관이 몸에 박힌 도구에서, 사용자는 10% 나은 것으로 갈아타지 않는다 — 기존 도구가 **원리적으로 못 하는 일**이 있어야 갈아탄다. 그리고 열광적 소수의 사랑은 대중 전환의 예측 변수가 아니다.

## 문제 정의 / 목표

포화된 할일 앱 시장 진입. 목표는 세 사례의 패턴 이식/기각으로만 도출된 전략 — 특히 P-C(실패 유비)를 통과하지 못하는 전략은 채택 금지.

## 패턴의 이식/기각 판정

- **P-C(반패턴)를 필터로 먼저 설치:** "우리 안(案)은 기존 앱 사용자가 *갈아타야만 얻는* 무엇을 주는가? 그것은 재해석(더 예쁘고 빠른 같은 것)인가, 아니면 기존 도구가 원리적으로 못 하는 것인가?" — 모든 후보 전략이 이 질문을 통과해야 한다.
- **P-A 이식:** 할일 데이터를 로컬 플레인 파일로(소유권), 플러그인으로 워크플로를 사용자가 조립(확장성). 기존 지배자(Todoist·TickTick)는 서버 DB + 폐쇄 기능셋이라 **구조적으로 따라올 수 없는 축**이다(그들의 비즈니스 모델이 서버 종속에 걸려 있으므로) — P-C 필터 통과: "내 데이터·내 규칙"은 재해석이 아니라 지배자가 원리적으로 못 주는 것.
- **P-B 이식(수정):** 표적을 "할일이 많은 사람"이 아니라 **"자기 생산성 시스템을 직접 설계하는 사람"**(GTD/PARA 실천자, 개발자, 옵시디언 사용자층과 겹침)으로. 이들의 고통: 기성 앱의 고정된 워크플로가 자기 시스템과 안 맞음. 가격은 Superhuman식 고가 구독 대신 Obsidian식(개인 무료 + 동기화·팀 유료) — 이 계층은 구독 피로가 크고 소유 선호가 강하다는 판단으로 P-B의 가격 요소는 **부분 기각**.
- **교차 검증(사례 간 상호 필터):** P-A 전략을 P-B로 검증 — 니치가 지불하는가? (동기화·모바일이 지불점) / P-B 니치를 P-C로 검증 — 이 니치는 전환 비용을 넘는가? (이 계층은 도구 갈아타기 자체를 즐기는 얼리어답터라 전환 장벽이 시장 평균보다 낮다 — 통과).

**도출 전략:** "시스템을 직접 짓는 소수"를 위한 **로컬 파일 기반·플러그인 확장형 할일 도구.** 성장은 커뮤니티/플러그인 생태계 주도, 수익은 동기화+상업 라이선스. 대중 정면전 금지(P-C).

## 명시적 가정 목록 (틀렸을 경우의 영향 포함)

- **가정 1 (유비 타당성):** 노트 시장의 P-A가 할일 시장에서도 작동한다 — 즉 할일에서도 "시스템을 짓는 소수"가 존재하고 충분히 크다. → 틀리면 전략 전체 붕괴. 검증: 기존 생태계의 대리 신호(옵시디언 태스크 플러그인 사용량, GTD 커뮤니티 규모)를 착수 전에 조사.
- **가정 2:** 지배자들이 로컬 파일·개방 생태계로 못 따라온다(비즈니스 모델 구속). → 틀리면(따라옴) 해자 소멸 — 다만 그들의 서버 수익 구조상 가능성 낮다고 평가.
- **가정 3:** 플러그인 생태계가 임계 질량에 도달 가능(초기 확장 API에 개발자가 붙는다). → 틀리면 P-A의 절반(확장성)이 죽음 — 초기 10개 핵심 플러그인은 자체 제작으로 마중물.

## 접근 및 단계

1. **유비 검증 조사(착수 게이트):** 가정 1의 대리 신호 조사 — 신호 없으면 이 전략을 폐기하고 유비 수집부터 다시.
2. **코어 = 파일 포맷 + 최소 앱:** 플레인 텍스트 할일 포맷(기존 마크다운 생태계와 호환) + 빠른 데스크톱 앱. P-A의 소유권 축부터.
3. **확장 API + 마중물 플러그인 10종:** 생태계의 시동(가정 3 대응).
4. **커뮤니티 채널 개설과 공개 개발:** P-A의 성장 엔진. 마케팅비 대신 개발 과정 자체를 콘텐츠로.
5. **수익화(동기화·모바일)는 거점 확보 후:** Obsidian의 순서를 따른다 — 신뢰(로컬·무료) 먼저, 과금(부가 서비스) 나중.

## 대안과 기각 사유

- **"더 예쁘고 빠른 할일 앱"(재해석 전략):** 기각. P-C 필터 정면 위반 — Arc의 길. 열광적 리뷰와 낮은 전환율이 공존하다 소진된다.
- **AI 기능 차별화(자동 분류·우선순위 제안):** 기각. P-C 필터 — 지배자가 다음 분기에 복제 가능(원리적 배타성 없음). 기능은 해자가 아니다.
- **B(카테고리 통합, Notion의 길):** 기각. 그 자리는 이미 Notion이 있고, 통합 축의 후발은 원조와의 정면전이다. (유비의 규칙: 이미 점유된 패턴의 재이식은 유비가 아니라 모방.)
- **대중 무료 + 광고/데이터 수익:** 기각. P-A의 소유권 가치와 정면 모순 — 패턴들끼리의 정합성도 검사 대상이다.

## 리스크 & 완화

- **R1: 유비 오류(가정 1) — 할일 도구는 노트만큼 '시스템 구축욕'을 자극하지 않을 가능성.** → 착수 게이트(1단계)에서 대리 신호 미달 시 진입 자체를 재고 — 유비 전략의 첫 산출물은 제품이 아니라 유비의 검증이어야 한다.
- **R2: 니치의 천장(시장이 작아 사업 미성립).** → Obsidian의 실증(니치로 시작해 수백만 사용자)이 반례지만, 할일 도메인에서 재현 보장은 없음 — 인접 확장 경로(노트·프로젝트 관리와의 결합)를 지도만 그려둠.
- **R3: 오픈 생태계의 품질 관리(악성/저질 플러그인).** → 서명·리뷰 프로세스 — Obsidian이 이미 밟은 길을 그대로 유비.

## "완료"의 판정 기준

- 전략의 모든 채택/기각이 P-A/P-B/P-C 중 하나로 소급된다(패턴 밖 임의 결정 0).
- **P-C 필터 통과 증명**이 문서화돼 있다: 우리가 주는 것이 지배자가 원리적으로 못 주는 것임의 논증.
- 유비의 핵심 가정(1번)이 착수 전 대리 신호로 검증됐다.
- 실패 유비(Arc)에서 추출한 반패턴이 기각 사유로 실제 사용된 흔적이 있다(최소 2회 — 실제로 재해석 전략과 AI 전략을 죽였다).
