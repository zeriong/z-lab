# 가입 첫 주 이탈률 60% — 원인 조사
- 추론 프레임: hypothesis-loop
- 한 줄 요약: 이탈은 관찰된 사건이 아니라 "재방문의 부재"라는 집계다. 부재는 원인을 말하지 않으므로, 60%를 단일 원인의 크기로 읽는 순간 조사는 실패한다 — 기각 조건을 데이터 열람 전에 등록한 가설 원장을 세우고, 기존 데이터로 되는 싼 실험부터 병렬로 돌려 60%를 원인 성분으로 분해한다.

## 프레임이 요구하는 시작점 — 관찰 사실과 추측의 분리

이 프레임의 실패 모드 두 개를 먼저 심문한다. (a) **조기 수렴**: 팀에는 이미 "온보딩이 복잡해서"라는 서사가 있고, 이 조사는 그 서사의 확인 작업으로 전락하기 쉽다. (b) **가설 없는 계측 살포**: "일단 모든 이벤트를 심자"는 4주 뒤 흥미로운 그래프와 결정 불가 상태를 낳는다. 이 플랜의 구조 전체가 이 둘의 방지 장치다.

**관찰 사실 (검증 대상 포함):**
- F1: 대시보드가 "가입 후 7일 내 재방문 없음 = 60%"를 표시한다. 단, 우리가 관찰한 것은 60%라는 숫자이지 이탈이 아니다. '방문'으로 집계되는 이벤트의 정의, ID 체계, 봇·중복 가입 처리 여부는 미확인 — **계측 자체가 용의자 명단에 오른다.**
- F2: 이 60%가 "언제부터"인지 모른다. 항상 그랬는지, 특정 시점에 악화됐는지에 따라 탐색 공간이 완전히 달라진다.
- F3: 이탈자 본인이 남긴 이유 데이터는 0건이다. 떠난 사람은 아무 말 없이 사라졌다.

**추측 (사실인 척 유통되는 것들 — 전부 가설로 강등):** "온보딩이 복잡하다", "가격 때문이다", "타깃이 아닌 유저가 유입된다". 근거 데이터가 제시된 적 없으므로 원장에서 다른 가설과 동등하게 취급한다.

**층위 이동 — 질문의 좌표를 바꾼다.** "왜 떠나는가"(집계에 대한 단수형 질문)를 "떠난 각 사용자의 **마지막 관측 행동**은 무엇인가"(개별에 대한 분포 질문)로 전환한다. 이 전환이 메타 가설을 낳는다: **60%는 하나의 원인이 아니라 여러 세그먼트의 합일 수 있다.** 단일 원인 전제 자체를 검증 대상(H0)으로 등록한다.

## 문제 정의 / 목표

1차 목표는 개선이 아니라 **원인 특정 = 재현 레시피**다: "조건 X를 가진 신규 가입자는 확률 Y로 이탈한다"를 분석에 쓰지 않은 새 코호트에서 사전 예측으로 적중시키는 것. 개선안은 이 조사의 산출물이 아니며, 원인 명세서에 딸린 "다음 실험 후보" 목록으로만 남긴다. 조사 기간 3주, 분석 인력 1~2인 가정.

## 명시적 가정 목록

- **A1: 사용자 단위로 조인 가능한 이벤트 로그가 최소 8주치 존재한다.** 틀리면 → 1주차의 싼 실험 대부분이 불가능하고, 계측을 먼저 심고 데이터가 쌓이길 기다리는 2~4주가 선행된다. **이 플랜의 실행 가능성 전체가 이 가정에 걸려 있다 — 착수 첫 반나절에 실측 확인.**
- A2: 60%를 만든 대시보드 쿼리를 우리가 재현할 수 있다. 틀리면 → 조사 대상이 유령이 된다. 재현 불가 자체가 H3(계측 결함)의 강력한 증거로 승격.
- A3: 잔존 40%를 비교군(성공 사례)으로 쓸 수 있다. 틀리면(잔존자도 저활성 좀비라면) → 잔존/이탈 비교 설계가 붕괴, 비교 기준을 "핵심 행동 빈도 상위 사용자"로 교체.
- A4: 조사 3주간 제품·유입 캠페인에 큰 변경이 없다(관찰 기간 동결). 틀리면 → 코호트가 오염되므로 변경 시점 전후로 코호트를 분리해 분석.

## 가설 원장 — 기각 조건은 데이터보다 먼저

각 행: 가설 / **선등록 기각 조건**(데이터 열람 전 타임스탬프 기록, 사후 수정 금지) / 실험 / 비용 등급(①기존 데이터만 ②새 계측 필요 ③사용자 접촉 필요).

- **H0 (혼합 분포):** 60%는 2개 이상의 이질적 세그먼트의 합이다. — 실험: 이탈자를 마지막 관측 행동·도달 단계로 군집화. — 기각: 단일 지점이 이탈자의 80% 이상을 설명하면 "단일 병목"으로 판정, H0 기각. — 비용 ①.
- **H1 (활성화 실패):** 첫 세션에서 핵심 가치 행동에 도달하지 못한다. — 실험: 잔존 40% vs 이탈 60%의 첫 세션 핵심 행동 도달률 비교. — 기각: 두 집단의 도달률 차이가 5%p 이내면 기각(도달해도 떠난다는 뜻이므로). — 비용 ①.
- **H2 (기대 불일치 / 유입 품질):** 유입 채널이 약속한 것과 제품이 다르다. — 실험: 이탈률을 채널·캠페인 좌표로 재절단. — 기각: 채널 간 이탈률 편차가 ±10%p 이내(전 채널 균일)면 기각. — 비용 ①.
- **H3 (계측 결함):** 60%의 상당분은 이탈이 아니라 측정 오류다(웹→앱 이동 시 ID 분리, 미인증 봇, 중복 가입, '방문' 이벤트 정의 누락). — 실험: 이탈자 무작위 200명 수동 감사 — ID 교차 매칭, 이메일 도메인, 결제·지원 티켓 대조. — 기각: 표본 오분류율 5% 미만이면 계측 무죄. — 비용 ①~②.
- **H4 (세션 사이의 공백):** 첫 세션은 성공하지만 돌아올 트리거가 없다 — 이탈은 세션 '안'이 아니라 **세션 '사이'**에서 일어난다. 단계별 퍼널만 보면 이 구간은 아무도 모델링하지 않는 틈이다. — 실험: 핵심 행동 도달자만 추려 그중 이탈 비율 산출 + 복귀 접점(이메일 발송·오픈) 로그 대조. — 기각: 핵심 행동 도달자의 이탈률이 25% 미만이면 주성분 아님으로 기각. — 비용 ①.

**좌표계 회의주의(별도 실험 E0, 비용 ①):** 60%를 주간 가입 코호트 시계열, 가입 시각(로컬 타임존), 기기/플랫폼, 릴리스 버전 좌표로 각각 재투영한다. 특정 릴리스 이후 급등이면 원인 공간이 극적으로 줄고, 균일하면 구조적 원인 쪽 가설(H1, H4)에 가중치가 실린다. E0은 가설 판정용이 아니라 **가설 생성용** — 여기서 튀는 조각은 다중 비교 착시일 수 있으므로 발견 즉시 승격 금지, 새 가설로 원장에 등록만 한다.

## 접근 및 단계

1. **0단계 (반나절):** A1·A2 실측 — 로그 존재·조인 가능성 확인, 대시보드 쿼리 재현. 검증: 재현된 숫자가 60%±2%p.
2. **1주차:** 비용 ① 실험 전부(E0, H0~H4) **병렬** 실행. 단, 해석 순서 규칙: **다른 실험 결과의 해석은 H3 표본 감사 결과 확인 후에만** — 계측이 유죄면 나머지 데이터도 오염돼 있기 때문이다. 검증: 각 가설에 승격 후보/기각/판정 불가 라벨.
3. **2주차:** 생존 가설에 **프레임 재귀 적용** — 예: H1 생존 시 "왜 도달 못 하는가"의 하위 원장(H1a: UI 마찰 지점, H1b: 콜드스타트—보여줄 데이터가 비어 있음, H1c: 권한·연동 장벽)을 같은 규율(선등록 기각 조건)로 작성·검증. 필요 시 이 시점에만 비용 ② 계측을 해당 가설이 요구하는 최소 범위로 추가.
4. **3주차:** 승격 심사 — 생존 가설로 "조건 X → 이탈 확률 Y" 예측문을 작성하고, **분석에 쓰지 않은 최신 주간 코호트**에 적용해 적중 여부 실측. 검증: 예측 적중 시 원인 명세서 확정, 실패 시 가설 강등.

**판단의 규칙화 (게이트 3개, 그 이상 만들지 않는다):**
- **수렴 차단기:** 단일 가설에 총 분석 시간의 40% 초과 투입 금지. 초과 시 강제로 다음 가설로 전환. (조기 수렴 방지의 기계화)
- **승격 게이트:** "원인"이라는 단어는 새 코호트 사전 예측 적중 없이는 사용 금지. 사후 설명은 몇 개든 만들 수 있다 — 예측만이 재현 레시피다.
- **기본값 게이트:** 2주차 종료까지 아무 가설도 승격 후보가 못 되면 → 기본값 발동: "기존 데이터로는 특정 불가"를 공식 출력으로 문서화하고 비용 ③(이탈자 표적 인터뷰 10명)으로 전환. 결론 없음도 게이트의 유효한 출력이다.

## 대안과 기각 사유

- **즉시 개선 실험(온보딩 개편 A/B)부터:** 기각. 원인 미상 상태의 개선은 가설 하나(그것도 팀이 원래 믿던 것)에 올인하는 도박이며, 프레임의 실패 모드 그 자체다. **부활 조건:** 승격 게이트를 통과한 원인이 나오면, 그 원인을 겨냥한 A/B가 바로 다음 단계가 된다.
- **전면 계측 강화부터:** 기각. 가설 없는 계측 살포는 두 번째 실패 모드다. **부활 조건:** 특정 가설이 판정 불가로 끝나고 그 가설이 요구하는 최소 계측이 명시된 경우, 그 범위만.
- **이탈자 대규모 설문:** 기각(1차에서). 이미 떠난 사람은 응답하지 않으므로 생존 편향이 구조적이고, 비용 ③이다. **부활 조건:** 기본값 게이트 발동 시 표적 인터뷰 형태로 축소 부활.

## 리스크 & 완화

- **R1: 조기 수렴 — "온보딩 탓" 서사의 중력.** → 기각 조건 선등록(사후 해석 봉쇄) + 수렴 차단기 + 불편한 가설(H2, H3)의 원장 강제 포함.
- **R2: 계측 결함이 조사 자체를 오염(H3 참이면 H0~H4의 데이터도 오염).** → 해석 순서 규칙으로 H3 감사를 해석의 전제 조건으로 배치.
- **R3: 다중 비교 착시 — 좌표를 여러 개 자르면 우연히 튀는 조각이 반드시 나온다.** → 발견은 가설 생성용, 승격은 오직 새 코호트 예측 적중으로만(승격 게이트가 겸임).
- **R4: 운영 부담 — 원장과 게이트가 1~2인 팀을 짓누름.** → 원장은 표 1장, 게이트는 3개로 상한 고정, 기각 조건은 각 1문장. 이 문서의 규칙 수 자체를 감사했다.

## "완료"의 판정 기준

- **실측 판정문:** 조사 종료 시점에, 분석에 사용하지 않은 최신 주간 가입 코호트에 대해 사전 작성된 예측문("조건 X인 가입자는 이탈한다")이 **해당 코호트 이탈자의 50% 이상을 적중**하거나, 그렇지 못하면 "기존 데이터로 특정 불가"가 기본값 게이트의 출력으로 문서화되어 있다 — 둘 중 하나가 반드시 성립.
- 60%가 원인 성분별로 분해된 표가 존재하고, 각 성분에 승격/기각/판정 불가 라벨과 근거 실험이 연결되어 있다.
- 모든 기각 조건에 데이터 열람 이전의 타임스탬프가 남아 있다(선등록의 물증).
- 산출물에 개선안이 없다 — 있다면 이 조사는 목표를 침범한 것이다. 개선 실험 후보 목록은 원인 명세서의 부록으로만 존재한다.
