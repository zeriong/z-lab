# 스타트업 웹사이트 리뉴얼: 페이지 절반으로
- 추론 프레임: delete-first
- 한 줄 요약: 페이지를 하나씩 깎지 않는다. "방문자가 해야 할 단 하나의 행동" 문장을 판결 기준으로 세우고, 정보구조를 탐색형에서 경로형으로 바꾸는 구조적 결정 하나로 페이지 대부분을 한꺼번에 처형한다 — 절반은 목표가 아니라 상한이 된다.

## 프레임이 요구하는 시작점 — 후보 전부 깔고, 한 문장으로 처형 재판

delete-first의 실패 모드는 두 가지다. ① 삭제를 정당화한 전제(어떤 방문자를 버렸는가)를 암묵에 두는 것 — 공격적으로 지울수록 전제 공개가 무거워져야 한다. ② 레버 없이 페이지를 하나씩 깎는 것 — 개별 협상은 반드시 정치가 된다. 이 플랜의 방어: 판결 기준 문장에 타깃 방문자를 박아 넣고(전제가 문장 안에 있으므로 암묵화 불가), 개별 판결에 앞서 여러 페이지를 동시에 불필요하게 만드는 구조적 결정을 먼저 찾는다.

**처형 기준 문장:** "타깃 방문자 — 도입을 검토하는 실무 리드 — 가 데모를 신청한다."

이 문장의 필요조건임을 증명하지 못하는 페이지는 유죄다(입증 책임은 페이지 쪽에 있다). 지원자·투자자·언론은 이 문장에 없다 — 그들을 버렸다는 것이 이 리뉴얼의 대가이며, 아래 가정 1에 굵게 노출한다.

**숫자 전 의미 고정 — "절반"의 단위.** 페이지 = 검색엔진에 인덱싱되는 독립 URL이자 우리가 유지보수하는 문서. 법적 의무 페이지(개인정보처리방침·이용약관)는 재판 면제, 카운트 제외. 이 정의가 없으면 14페이지를 15,000px 원페이지 하나로 이어 붙여 "1페이지 달성"을 선언하는 숫자 게임이 가능하다 — 병합은 유지보수 표면이 함께 줄 때만 삭제로 인정한다.

**판결 결과: 독립 URL 3개(경로형 홈 / 가격 / 데모 신청) + 조건부 1개(블로그).** "절반(7)"은 목표였지만 문장 재판의 판결은 3~4다 — 절반은 상한으로 재해석한다. 보류 판결은 금지한다. 보류는 정치의 은신처다 — 판단이 안 서는 페이지는 지금 조건부 판결(블로그처럼 임계값 내장)로 변환한다.

## 삭제 레버 — 탐색형에서 경로형으로 (층위 이동)

페이지를 하나씩 재판하기 전에, 여러 페이지를 한꺼번에 불필요하게 만드는 결정이 있다: **사이트를 "페이지의 집합(방문자가 내비게이션으로 고른다)"에서 "반론의 순서 있는 해소(방문자가 하나의 서사를 따라 내려간다)"로 재정의한다.** 페이지 목록이 반론 목록으로 바뀌는 층위 이동이다. 실무 리드가 데모 신청 전에 품는 반론은 다섯 가지로 수렴한다고 본다 — ① 이게 내 문제를 푸나(제품) ② 얼마인가(가격) ③ 믿을 만한가(증거·팀) ④ 도입이 귀찮지 않나(보안·연동) ⑤ 경쟁 제품 대비 왜 이건가. 경로형 홈은 이 반론들을 순서대로 해소하고 데모 신청에서 끝나는 단일 서사다. 이 결정 하나가 기능 3페이지·FAQ·고객사례·파트너를 동시에 페이지에서 섹션으로 강등시켰다 — 레버가 먼저, 개별 재판은 그 다음이었다.

## 문제 정의 / 목표

14페이지로 비대해진 스타트업 웹사이트를 리뉴얼하며 페이지를 절반 이하로 줄인다. 진짜 목표는 페이지 수가 아니라 **방문자 행동의 집중** — 페이지 수는 행동이 분산됐다는 증상이다. 성공은 "적은 페이지"가 아니라 "삭제 후에도 데모 신청 전환율이 유지·상승"으로 판정한다.

## 명시적 가정 목록

- **가정 1 (전체가 걸린 전제): 타깃 방문자는 단일 페르소나 — 도입을 검토하는 실무 리드다. 지원자·투자자·언론은 이 사이트의 고객이 아니다.** 틀렸을 경우: 채용·팀·뉴스의 처형 판결이 전부 무효. 이 삭제 전체가 이 한 줄에 걸려 있으므로 판결표 상단이 아니라 처형 기준 문장 자체에 박았다. 보험은 부활 게이트(아래).
- 가정 2: 단일 핵심 행동은 "데모 신청"이다(세일즈 주도 B2B). 틀렸을 경우(셀프서브 가입으로 전환 등): 기준 문장을 교체하고 재판을 전면 재심 — 단, 판결표 구조 덕에 재심 비용은 재채점 수준으로 낮다.
- 가정 3: 블로그의 오가닉 유입이 데모 신청에 기여하는 비중은 5% 미만이다. 틀렸을 경우: 조건부 판결에 내장된 대로 블로그 자동 생존 — 판결이 뒤집혀도 플랜은 안 깨진다.
- 가정 4: 실무 리드의 반론은 5개 이내로 수렴한다. 틀렸을 경우: 경로형 홈이 비대해져 탐색형 압력이 재발 — 섹션 상한 6개 규칙(리스크 R4)으로 조기 감지.

## 접근 및 단계

1. **기준선 측정(2주):** 애널리틱스·서치콘솔로 페이지별 유입과 데모 신청 경로 참여율 측정. → 검증: 페이지별 "데모 기여율" 표 완성, 블로그 조건부 판결의 임계값(5%) 판정 가능 상태.
2. **처형 재판 확정:** 판결표의 판결·근거·대체재·부활 조건 4열을 채우고 이해관계자 서명. → 검증: 14개 전 항목 4열 완비, 보류 0건.
3. **인바운드 링크 전수 조사(사이 탐색):** 삭제의 사고는 페이지가 아니라 페이지 *사이*에서 터진다 — 외부 백링크, 세일즈 이메일 서명, 집행 중인 광고의 랜딩 URL, 인쇄물 QR까지 죽는 URL로의 유입 경로를 전수 조사하고 301/410 맵 작성. → 검증: 크롤 시뮬레이션에서 내부 404 0건, 광고 랜딩 URL 교체 완료.
4. **경로형 홈 구축 + 재귀 재판:** 반론 섹션 최대 6개로 제작. 살아남은 데모 신청 폼에도 같은 프레임을 재귀 적용 — 폼 필드 하나하나가 "제출"이라는 행동의 필요조건인지 재판(이름·회사 이메일·용건 3개만 생존, 전화번호·직급·규모는 처형·세일즈 첫 통화로 이관). → 검증: 폼 필드 ≤ 3, 외부인 5분 사용성 테스트에서 데모 신청 도달.
5. **배포 + 4주 관측 + 게이트 가동:** 전환율·404·오가닉 추이 관측. 신규 페이지 요구는 게이트로만 통과 — "새 페이지는 기준 문장의 필요조건 증명과 판결표 등재를 지참해야 하며, 기본값은 기각." 부활 심사는 분기 1회, 부활 조건 수치를 넘지 못하면 자동 기각(기본값 있는 게이트만이 작동하는 게이트다). → 검증: 아래 완료 판정.

## 대안과 기각 사유

- **시각 리디자인만(정보구조 유지):** 기각. 페이지 수는 증상이고 원인은 행동의 분산 — 시각 교체는 원인에 손대지 않는다. 부활 조건: 처형 재판 후에도 전환율이 안 오르면 병목이 구조가 아니라 카피·시각이라는 증거이므로 그때 수행.
- **트래픽 낮은 순 점진 폐기:** 기각. 레버 없는 하나씩 깎기이며, 트래픽은 필요조건의 척도가 아니다(트래픽 1위 블로그가 데모 기여 0일 수 있다). 부활 조건: 조직이 판결표 합의에 실패하면 차선책으로.
- **전부 원페이지 통합:** 기각. "절반"의 단위 정의가 금지하는 숫자 게임 — URL은 줄지만 유지보수 표면이 안 준다. 가격은 검색 의도("제품명 가격")가 독립적이라 별도 URL의 실익도 있다.

## 리스크 & 완화

- **R1 처형된 페르소나의 은닉 수요:** 지원자가 사이트를 보고 지원을 결심해 왔을 수 있다. → 분기 부활 심사에서 채용팀의 지원자 유입 데이터를 정식 안건으로. 가정 1의 보험.
- **R2 SEO 손실:** 블로그·사례 삭제로 오가닉 하락. → 1단계 측정이 사전 방어(기여 5%↑면 자동 생존), 301 맵 + 4주 관측이 사후 방어.
- **R3 페이지 재증식:** 리뉴얼 후 "이 페이지 하나만"이 반복되면 3년 뒤 원점. → 5단계의 신규 페이지 게이트를 상설화. 게이트·심사 모두 분기 1회 수준으로 설계해 운영 부담이 삭제의 이득을 갉아먹지 않게 한다.
- **R4 경로형 홈의 비대화:** 반론이 늘며 무한 스크롤화. → 섹션 상한 6개 고정, 초과 요구는 "어느 기존 반론 섹션을 대체하는가"를 지참해야 통과.

## "완료"의 판정 기준

- **서치콘솔 인덱싱 보고서 기준, 자체 도메인의 인덱싱 URL 수가 리뉴얼 전 대비 50% 이하다(법적 페이지 제외 기준으로 실측).**
- **리뉴얼 후 4주 데모 신청 전환율(세션 대비)이 리뉴얼 전 4주 기준선 이상이다** — 이 문장이 "삭제가 손실이 아니었다"의 실측 증거이며, 미달 시 부활 심사를 즉시 소집한다.
- 서버 로그 기준, 삭제 URL로의 외부 유입 404가 0건이다(전부 301/410 처리 확인).
- 판결표 14개 항목 전부에 판결·근거·대체재·부활 조건이 기록되어 있고, 문서 첫 줄에 가정 1(버린 방문자)이 명시되어 있다.
