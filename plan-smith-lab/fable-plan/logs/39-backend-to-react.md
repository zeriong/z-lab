# 로그 — 39 백엔드 개발자의 React 전환 (misconception-first) [Fable]

## 진입 시 사고
지시가 프레임의 절반을 이미 해줬다 — 오개념 후보(요청-응답, 동기 흐름, 상태는 DB에)와 봉인 목록의 예시까지 명시돼 있었다. 그래서 진입 질문은 "오개념이 뭔가"가 아니라 **"오개념을 어떤 축으로 정렬하고, 사망을 어떻게 판정하는가"**가 됐다. frames.md가 적시한 실패 모드 — 목록을 만들고 결국 난이도순으로 가르치는 회귀 — 를 막으려면 난이도를 대체할 정렬 기준이 필요했고, "오염 반경"(하류 개념을 몇 개 조용히 오염시키는가)을 세웠다. M1(렌더=요청당 1회 응답)이 최상위에 오는 건 이 축에서만 정당화된다 — 난이도로는 M1이 제일 쉬운 축에 속한다. 전제조건 검사는 required component라서 넣은 게 아니라, 가정 1(백엔드 습관이 굳어 있고 프론트 백지)이 틀리면 지도 전체가 무효라는 구조 때문에 서문이 아닌 0주차 관문으로 승격시켰다.

## 프레임이 만든 결정적 순간
- **진도 단위의 교체.** "주차는 예산, 관문은 사망 확인서"라는 층위 선언이 이 플랜의 등뼈다. 사망에 조작적 정의(실행 전 예측 적중 + 구모델의 오답 재현 설명)를 붙이지 않으면 "이해했다"는 자기 보고로 회귀하므로, 완료 판정까지 같은 형식(제3자 출제 충돌 문제 5개 중 4개 적중)으로 연장했다.
- **M5의 배치.** 프레임을 기계적으로 따르면 M5는 useState를 배우는 1~2주에 놓인다(그때 잠복하므로). 하지만 오개념은 발화해야 충돌이 성립한다 — 그래서 "배우는 시점이 아니라 발화하는 시점"인 5주 데이터 페칭에 놓고, 5주차를 서버 상태 라이브러리 없는 손코딩으로 설계해 발화를 강제했다. 봉인 목록의 React Query 항목은 이 결정의 반대면이다(조기 도입 = M5 은폐).
- **봉인 근거의 통일.** 봉인 4건을 "어려워서 나중"이 아니라 은폐성(오개념이 죽기 전 도입하면 그 오개념을 가림)으로 통일했다. 난이도라는 단어를 정렬에서 추방한 결정이 봉인 섹션까지 관철된 지점.
- **6주 재귀 적용.** 교정 자체가 2차 오개념(파생값까지 전부 state로, 이펙트 공포)을 낳는다는 관찰을 예정된 회수 패스로 만든 것. frames.md의 required components에는 없는 확장이다.

## 프레임 이탈 / 관성 회귀 지점
- **1~3주의 화물 배열이 정석 커리큘럼과 거의 겹친다.** JSX→이벤트·폼→리스트는 어느 튜토리얼에나 있는 순서다. 각 주의 뼈대는 충돌 과제(M1~M3)라고 주장하지만, M2와 M3의 상대 순서가 오염 반경으로 강제되는지 화물의 자연 순서를 따라간 건지 플랜 본문은 구분해 주지 못한다. 실패 모드 ①이 화물의 이름을 달고 부분 회귀했을 가능성.
- **"오염 반경"이 조작화되지 않았다.** 위계의 유일한 정렬 기준인데 측정 문장이 없다 — 두 사람이 독립적으로 M1~M5를 같은 순서로 배열할 수 있는지 보장이 없고, 결국 저자의 직관이 축의 이름만 바꿔 단 것일 수 있다.
- **R4에서 프레임의 한계를 자백했다.** 오개념 뼈대는 무간섭 지식(라우팅, TS)을 다룰 언어가 없어 "화물"이라는 프레임 외 개념을 발명해 부착했다. 정직한 처리지만, 플랜의 상당 분량이 프레임 바깥에서 온다는 뜻이기도 하다.

## 방향성
"판단을 사전에 룰로 바꿔라"가 반복 축이다 — 예산 규칙(이월 1회, 꼬리 절삭), 스킵 게이트(기본 동작 명시), 봉인 파기 유혹 룰(3회 누적 시 심사). 실행 시점의 재량을 저자 시점의 규칙으로 앞당기는 습관. 유혹의 빈도를 해제 조건의 데이터로 쓰는 것은 min-diff의 temptation log 수법의 전용이다.

## 자가 평가
프레임 충실도: 상. required components 3종(전제조건 검사, 봉인 목록+해제 조건, 예측→충돌→재구성)이 전부 실체로 존재하고, 각각 장식이 아니라 판정 장치(관문, 해제 심사, 사망 확인서)로 기능한다. 흠: ① 오염 반경이 선언만 되고 조작화되지 않아 정렬의 재현 가능성이 없음, ② 1~3주가 화물 순서에 편승해 부분적 난이도순 회귀로 읽힐 여지, ③ 주 7시간 예산에 충돌 과제+확인서+유혹 기록+R3의 기록 상한 관리까지 얹혀 기록 계층이 과중함 — R3으로 상한을 걸었지만 상한 관리 자체도 비용이다.
