# 레거시 모놀리스 인증 시스템 교체
- 추론 프레임: risk-matrix
- 한 줄 요약: 확률×피해 정렬이 드러낸 사실 — 상위 2개 리스크(전원 로그아웃, 세션 불일치)는 인증의 실패가 아니라 "두 시스템을 병행·동기화한다"는 문제 표현의 산물이다. 표현을 "세션 원본의 소유권 이관"으로 바꿔 상위를 구조로 죽이고, 남는 보안 리스크는 중단선과 만료 하드코딩으로 묶는다.

## 프레임이 요구하는 시작점 — 척도의 조작화, 정렬, 컷라인, 예외 1회

이 프레임의 실패 모드는 둘이다: 조작 정의 없는 척도가 연극이 되는 것, 그리고 "어려우니까 나중에"가 순위를 몰래 바꾸는 것. 전자는 척도를 채점 가능한 문장으로 고정해 막고, 후자는 위상 예외 1회 규칙과 "난이도는 순위를 못 바꾼다 — 난이도를 낮추는 작업이 그 항목의 1단계가 될 뿐"이라는 규칙으로 막는다.

**피해 척도 — 복구 주체로 조작화** (두 사람이 독립 채점해 일치해야 척도다):
- 5: 보안 경계 침해. 본인 아닌 주체의 타인 계정 인증 성공이 1건이라도 확정. 발생 자체가 비가역.
- 4: 복구에 사용자 행동 필요. 30% 이상이 재로그인·비밀번호 재설정을 해야 복귀.
- 3: 운영자가 복구 가능하나 사용자 체감. 1% 이상에서 로그인 실패나 세션 단절.
- 2: 사용자 체감 없는 내부 지표 악화(지연·비용).
- 1: 운영 불편.

**확률 척도 — 기준 시제를 고정한다**: "대책 없이 관성적 방식(구/신 병행 운영 + 저장소 동기화 + 일괄 컷오버)으로 진행할 때 마이그레이션 기간 내 발생"으로 채점한다. 이 고정이 없으면 플랜이 낮춰 놓은 확률을 플랜의 근거로 쓰는 순환이 생긴다.
- 5: 구조적 확정(설계의 논리적 귀결). 4: 부분 실패·경합 등 정상 운영 사건만으로 발생. 3: 복수 조건의 동시 성립 필요. 2: 능동적 공격자 필요, 표면은 열려 있음. 1: 공격자와 내부 실수의 결합 필요.

**정렬** (시나리오 수집은 가용성/일관성/보안/운영/성능 카테고리 스윕으로 망라했고, 커버는 위에서부터만 — 망라율은 목표가 아니다):

| # | 시나리오 | 피해 | 확률 | 점수 |
|---|---|---|---|---|
| F1 | 전 사용자 강제 로그아웃 — 컷오버가 세션을 무효화 | 4 | 5 | 20 |
| F2 | 세션 이중화 불일치 — 구/신 저장소가 서로 다른 진실 | 3 | 4 | 12 |
| F3 | 마이그레이션 중 계정 탈취 — 토큰 혼용·다운그레이드·유령 검증 경로 | 5 | 2 | 10 |
| F4 | 인가 매핑 오류로 권한 상승 | 5 | 2 | 10 |
| F5 | 해시 이관 실패로 로그인 불가 누적 | 3 | 3 | 9 |
| F6 | 롤백 불가점의 무자각 통과 | 4 | 2 | 8 |
| F7 | 이중 수용 기간의 고착 — 교체가 끝나지 않음 | 2 | 4 | 8 |
| F8 | 이중 검증 오버헤드로 지연 증가 | 2 | 3 | 6 |
| F9 | 로그인 UX 변화로 지원 문의 증가 | 1 | 4 | 4 |

**컷라인 = 8점.** F8·F9는 의도적 미커버 — 대책을 설계하지 않고 관측만 한다(복구 주체가 우리이고 사용자 손실이 없다). 미커버는 누락이 아니라 결정이므로 점수와 함께 이 표에 남긴다.

**위상 예외(1회):** 1위 F1의 대책 "세션을 죽이지 않는 이관"은 2위 F2의 대책 "세션 원본 단일화"가 깔려야만 성립한다 — 원본이 하나여야 '이동 없는 이관'이 가능하다. 따라서 실행 순서에서 F2의 기제를 최우선으로 올린다. 예외는 이것으로 소진되며, 이후 어떤 항목도 순서를 바꿀 수 없다.

## 층위 이동 — "교체"가 아니라 "원본의 소유권 이관"

정렬이 드러낸 것: F1과 F2는 인증 로직의 실패가 아니라 **마이그레이션 방식의 표현이 제조한 리스크**다. 세션 데이터를 새 저장소로 "옮기면" F1이 확정되고(확률 5의 근거), 옮기는 동안 둘을 "동기화하면" F2가 내장된다(확률 4의 근거). 그래서 문제의 표현을 바꾼다:

> 세션 레코드는 이동하지 않는다. 신 인증 서비스가 기존 세션 저장소를 자기 원본으로 채택하고, 모놀리스는 인증의 소유자에서 신 서비스의 클라이언트로 강등된다. 옮겨지는 것은 데이터가 아니라 소유권이다.

효과: F1 확률 5→1(세션을 무효화할 사건 자체가 없음), F2 확률 4→1(원본이 하나면 불일치는 정의상 없음). 새 상위는 F3~F5가 되고, 이들은 구조가 아니라 규칙(중단선·만료·단일 구현)으로 묶는다. 재정의는 공짜가 아니다 — "신 서비스 다운 = 전체 인증 다운"이라는 새 결합을 만들며, 이는 리스크 섹션에서 같은 척도로 재채점한다.

## 문제 정의 / 목표

모놀리스에 흩어진 자체 인증(세션 관리·비밀번호 해시·인가 검사)을 독립 인증 서비스로 교체한다. 목표는 기능 동등성이 아니라 **마이그레이션 기간의 리스크가 이 매트릭스로 통제되는 상태**: 상위는 구조로 막고, 잔여는 중단선으로 묶고, 미커버는 기록으로 남긴다.

## 명시적 가정 목록

- **가정 1: 세션이 서버 측 중앙 저장소(DB/Redis류)에 있고, 신 서비스가 그 저장소를 원본으로 채택할 수 있다.** 틀리면(인메모리 스티키 세션 등) 소유권 이관 자체가 불가 — 층위 이동이 무효가 되고 "세션 외부화"가 0단계로 선행되어야 한다. 플랜 전체가 걸린 가정이므로 착수 1주차에 실코드로 판정한다.
- 가정 2: 구 해시 알고리즘을 신 서비스에서 재현할 수 있다. 틀리면 전원 비밀번호 재설정 = F1급 피해를 자발적으로 만드는 것이므로, 빅뱅 대안까지 포함해 전면 재평가.
- 가정 3: 모놀리스의 인증·인가 검사 지점은 유한하고, 정적 분석 + 2주 런타임 관측으로 전수 식별 가능하다. 틀리면 유령 검증 경로가 F3의 표면으로 남고, 제거 단계가 무기한 보류된다.
- 가정 4: 활성 세션의 90% 이상이 90일 내 자연 재발급을 경유한다. 틀리면 이중 수용 종료 시 잔존 사용자의 강제 로그아웃(축소판 F1) — 크기를 잔존 곡선 실측으로 확인하고, 크면 종료일을 사전 공지로 전환한다.

## 접근 및 단계 (각 단계의 커버 항목과 검증)

1. **소유권 이관 기반 [F2 — 위상 예외로 선행]** — 신 인증 서비스를 세우고 기존 세션 저장소를 원본으로 채택. 모놀리스의 세션 읽기/쓰기를 신 서비스 API 위임으로 전환. 검증: 2주 섀도 — 신 서비스가 검증 결과를 병행 산출해 구 로직 결과와 샘플 비교, 불일치율 0.1% 미만이어야 게이트 통과. 섀도는 관측 전용이라 사용자 응답에 무영향.
2. **세션 연속성 실측 [F1]** — 위임 전환 전후 24시간의 재인증 요구율이 평상시 대비 +0.2%p 이내인지 측정. 초과 시 중단선 발동.
3. **검증 경로 단일화 [F3]** — 난이도를 낮추는 작업이 이 항목의 1단계다: 모놀리스의 인증·인가 검사 지점 전수 인벤토리(정적 grep + 런타임 카운터 2주, 배치·관리자 화면·내부 호출 포함 — 인증은 미들웨어 "사이"에서 새는 법이다). 이후 구 토큰 수용 로직을 신 서비스 안에 단일 구현으로 두고(구 검증 코드는 위임 껍데기만 남김), 수용 만료일을 코드에 고정 — 만료 도달 시 기본 동작은 거부. 연장은 코드 변경·리뷰·배포를 요구하는 무거운 행위로 만든다 [F7 동시 커버].
4. **발급 경로 이관 + 점진 재해시 [F5]** — 로그인 경로를 직원 → 1% → 10% → 100% 카나리로 이관. 로그인 성공 시 구 해시 검증 후 신 해시 재저장. 게이트 기본값은 정지: "중단선 무위반 72시간 + 재해시 오류율 0.01% 미만" 충족 시에만 다음 비율로 진행.
5. **인가 섀도 비교 [F4]** — 카나리 기간에 동일 요청의 구/신 인가 결정을 샘플 비교. 불일치 중 '신이 더 허용' 방향은 건수 무관 즉시 게이트 봉쇄 — 권한 상승 방향은 확률과 거래하지 않는다.
6. **구 코드 제거 [F6]** — 모든 단계는 한 문장 롤백("위임 플래그 off로 구 로직 복귀")을 가진 것만 진행. 롤백 불가점은 단 하나 — 구 검증 코드 삭제 — 이며, 인벤토리 전 지점의 구 경로 호출 카운터가 2주간 0일 때만 통과한다.

**중단선 (전 단계 공통, 게이트의 기본값은 롤백):**
- 로그인 성공률이 기저선 대비 −0.5%p로 15분 지속 → 해당 단계 자동 롤백. 재진행은 원인 문서화 후에만.
- 섀도 불일치율 0.1% 초과 지속 → 다음 게이트 봉쇄.
- **보안 절대선: 타인 계정 인증 성공 1건 확정 → 전 단계 중단, 구 토큰 수용 즉시 종료, 활성 세션 전수 재검증. 피해 5는 확률과 거래하지 않는다.**

## 리스크 & 완화 — 생존자에 대한 재채점 (프레임의 재귀 적용)

층위 이동이 살려낸 구조를 같은 척도로 다시 채점한다:
- **R1 위임 결합: 신 서비스 다운 = 전체 인증 다운** (피해 3 × 확률 3 = 9, 컷라인 위). 교체 전에는 없던 단일 장애점이다. 완화: 모놀리스에 TTL 60초 이하의 검증 캐시 — 단 로그아웃·계정 차단은 즉시 무효화 전파. 캐시가 만드는 세션 잔존은 F3의 축소판이므로 TTL 상한은 성능이 아니라 보안 요건으로 고정한다.
- R2 섀도 비교의 부하와 오탐 (피해 2 × 확률 3 = 6, 컷라인 아래) — 샘플링으로만 수행하고 대책 없이 관측. 의도적 미커버로 기록.
- R3 만료 하드코딩의 경직성: 정당한 연장 사유가 늦게 발견될 수 있다. 완화하지 않고 수용한다 — 연장 비용(코드 변경)은 F7을 막는 의도된 마찰이다.

## 대안과 기각 사유

- **빅뱅 컷오버(점검창 일괄 전환):** 기각. F1 확률을 5로 확정하고, F3 발생 시 되돌아갈 단계가 없어 중단선이 무의미해진다. 부활 조건: 사용자 수백 명 이하 내부 시스템 + 전원 재로그인 공지가 허용되는 경우 — 그때는 이 플랜이 과잉이다.
- **양방향 세션 동기화(두 원본 병행):** 기각. F2를 구조에 내장하고, 충돌 해소 규칙이 곧 보안 결정이 되어 F3 표면을 넓힌다. 부활 조건: 규제·지리 분할로 원본 단일화가 물리적으로 불가능한 경우뿐.

## "완료"의 판정 기준

- **구 인증 코드 삭제 커밋 이후 2주간, 인벤토리에 등재된 전 검사 지점의 구 경로 호출 카운터가 0이다.**
- 이관 전후 각 4주의 로그인 성공률 차이 ≤ 0.2%p, 마이그레이션이 유발한 강제 재로그인 0건(이중 수용 자연 만료분은 0.5% 미만 실측).
- 재현 환경에서 레드팀이 이중 수용 구성에 토큰 혼용·다운그레이드 공격을 시도해 인증 우회 0건.
- 컷라인 아래 항목(F8·F9·R2)이 점수와 함께 "의도적 미커버"로 문서에 남아 있다.
