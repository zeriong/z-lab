# "회의가 너무 많다" — 진단과 개선
- 추론 프레임: observe-first
- 한 줄 요약: "많다"는 증상 보고이지 진단이 아니다. 2주 관찰로 병인을 갈라내되, 결정을 가르지 못하는 지표는 수집을 금지하고, 전략은 사전 확정된 분기표의 if–then으로만 존재하게 한다.

## 프레임이 요구하는 시작점 — 관찰 설계 (전략은 아직 없다)

이 프레임의 실패 모드부터 심문한다. observe-first는 서두름이 아니라 **낭비로 죽는다** — 4주 뒤 "흥미로운 데이터"만 쌓이고 결정 가능한 분기가 하나도 없는 상태. 그래서 이 플랜의 제1규칙: **어떤 결정을 가르는지 말하지 못하는 지표는 수집 금지(고아 지표 0).** 아래 모든 지표는 자신이 가르는 분기의 번호를 달고 있고, 분기의 임계값은 데이터를 보기 전인 지금 확정한다.

두 번째 심문: 측정할 것이 정말 '회의 수'인가. 아니다 — 층위를 한 번 옮긴다. 10명이 앉아 2명이 말하는 60분 회의는 "회의 1건"이 아니라 **발화 2인시 + 청중 8인시**다. 불만의 실체는 개수가 아니라 각자가 지불하는 비용이므로, 관찰 단위를 **회의 → 인시(person-hour) × 역할(발화자/청중)**로 바꾼다. 이 이동이 처방의 후보군을 바꾼다: 회의를 하나도 없애지 않고 참석만 줄이는 처방(B3), 팀이 아니라 특정인만 고치는 처방(B2)이 이 단위에서만 존재한다.

**의미 고정 (숫자보다 먼저):**
- 회의: 2인 이상의 동기적 업무 대화 5분 이상. 캘린더 등재 여부 무관(허들·즉석 호출 포함).
- 산출: 종료 시점에 기록으로 남은 결정 또는 담당자가 지정된 액션 아이템. 구두 합의는 산출이 아니다 — 두 사람이 독립 채점해 같은 점수가 나와야 지표다.
- 유형(상호 배타 5종, 주최자가 종료 시 태그 1개): 결정 / 정보전달(일방향) / 상태동기화 / 공동작업 / 관계·의례.
- 발화 분포: 화상회의 스피커 통계 또는 관찰자의 2분 버킷 주발화자 태깅. 무발화 참석 = 발화 버킷 0인 참석자.

**지표 → 수집 → 가르는 분기:**

| 지표 | 수집 방법 | 걸린 분기 |
|---|---|---|
| M1 인당 주간 회의 인시(중앙값·p90) | 소급 4주 캘린더 + 실측 2주 | B1·B2 |
| M2 캘린더 밖 회의 인시 비율 | 실측 2주, 하루 1회 1분 자가 로깅 | B0 |
| M3 유형 분포 | 주최자 종료 태그 | B3·B4 |
| M4 산출 0 회의 비율(정기/비정기 분리) | 종료 시 결정·AI 개수 기입 | B4 |
| M5 무발화 참석 인시 비율 | 샘플 10개 회의 | B3 |
| M6 조각화 — 하루 중 90분 이상 연속 자유 블록 수 | 캘린더 계산 | B5 |

**관찰 계획 자체의 재귀 감사:** 이 측정이 "회의 위에 얹힌 잡무"가 되면 자기모순이다. 인당 부담 상한 = 회의당 30초(태그 1개 + 숫자 1개) + 하루 로깅 1분. 발화 측정은 전수 금지, 샘플 10회의만. 상한을 초과하면 **부담을 늘리지 않고 지표를 깎는다.**

**베이스라인 동결:** 실측 2주간 새 회의 규칙·감축 캠페인·노미팅데이 등 일체의 개입 금지(자연 변동은 위반이 아니다). 관찰 자체가 개입이 되는 문제(보이면 줄인다)는 소급 데이터로 방어한다 — 지난 4주 캘린더는 원리상 관찰 효과가 0이므로, 실측 2주 총량이 소급 평균 대비 20% 이상 낮으면 "관찰 효과" 플래그를 세우고 총량 판정은 소급 데이터를 채택한다.

## 문제 정의 / 목표

팀 불만 "회의가 너무 많다"의 병인을 데이터로 분리하고, 병인에 맞는 처방 1개를 시행해 6주 내 개선을 실측한다. 목표는 회의 수의 감소가 아니라 **불만을 만든 비용의 감소** — 병인이 조각화라면 회의 수가 그대로여도 성공이다.

## 명시적 가정 목록

- **가정 1 (플랜 전체가 걸림): 병인이 자명하지 않다.** 총량 과다 / 부하 편중 / 무의미 참석 / 산출 부재 / 조각화 중 무엇인지 데이터 없이 확정할 수 없다. → 틀리면(한 병인이 압도적이면) 2주 관찰은 순수한 지연 비용이다. 그래서 1단계에 조기 판정 게이트를 둔다.
- 가정 2: 팀 캘린더 데이터에 접근 가능하고, 캘린더가 동기 회의의 대부분을 반영한다. → 틀리면 M1·M6의 신뢰가 무너짐. M2(캘린더 밖 로깅)의 가중치를 올리고 실측을 1주 연장.
- 가정 3: 팀에 2주를 견딜 여유가 있다(불만이 번아웃 위기 수준이 아니다). → 틀리면 관찰은 사치다. 응급 규칙 1개(정기회의 전원 옵셔널화)를 먼저 시행하고, 그 규칙을 동결 대상에 포함한 채 관찰한다. 전환 조건도 규칙: 팀원 과반이 요청하면.
- 가정 4: 샘플 10회의가 발화 분포를 대표한다. → 틀리면 B3만 "판정 불능"으로 기록하고 다른 분기는 유효하게 판정한다.

## 접근 및 단계

1. **소급 수집(3일):** 지난 4주 캘린더에서 M1·M6·정기회의 목록 추출. → 검증: 조기 판정 게이트 — 한 지표가 사전 임계의 2배를 넘으면 실측을 1주로 단축하고 해당 분기로 직행. 기본값은 "단축 없음"(기본값 없는 게이트는 게이트가 아니다).
2. **실측 2주:** 태깅·로깅·샘플 발화 측정, 동결 유지. → 검증: 태깅 누락률을 주 1회 점검, 30% 초과 시 태깅 주체를 주최자 1인으로 축소(부담 상한 규칙).
3. **판정(2일):** 분기표에 대입. → 검증: 두 사람이 데이터만 보고 **독립 판정**해 같은 분기에 도달해야 성립. 불일치는 지표 정의가 덜 고정됐다는 신호 — 정의를 고치고 재판정.
4. **처방 시행(반드시 1개만):** 복수 분기 점화 시 인시 절감 기대량이 최대인 것 하나만. 동시 개입 금지 — 효과 귀속이 불가능해진다. → 검증: 6주 후 동일 계기로 재측정.

**분기표 (임계는 지금 확정):**

- **B0 (정지 분기, 최상류):** M2 ≥ 30% → 공식 회의 개입 전면 보류. 캘린더 회의를 줄여봤자 즉석 호출로 새는 풍선 효과다. 처방은 비동기 채널이 왜 실패하는가의 조사로 전환. *순서 근거: 이 분기가 참이면 나머지 모든 처방의 표적이 허상이 된다.*
- **B2 (분포 판정, 두 번째):** p90/중앙값 ≥ 2.5 → 총량이 아니라 편중의 병. 상위 부하자의 역할 재설계(대표 참석·위임)만 하고 팀 전체 규칙 도입 금지. *순서 근거: 분포 문제에 전체 규칙을 처방하는 것이 이 도메인의 고전적 오처방 — 전원 판정보다 먼저 걸러야 한다.*
- **B1:** M1 중앙값 ≥ 주 12시간 → 총량의 병 → 정기회의 일몰제: 모든 정기회의는 6주 후 자동 만료, 갱신 조건은 직전 6주의 산출 기록 제시(기본값이 만료인 게이트 — 정기회의는 스스로 죽지 않는다).
- **B4:** 정기회의 중 산출 0 비율 ≥ 50% → 산출의 병 → B1과 동일한 일몰제로 수렴.
- **B3:** M5 ≥ 40% → 참석의 병 → 참석 기본값을 옵셔널로 바꾸고 결정 기록 열람으로 대체. 회의 수 불변, 청중 인시만 감축.
- **B5:** M6 < 2인데 M1은 정상 → 조각화의 병 → 총량 불변, 배치 개입(회의를 특정 요일·시간대로 집결). *총량 감축은 이 병에 무효 — 같은 조각이 더 흩어질 뿐이다.*
- **무점화:** 어떤 분기도 안 걸리는데 불만은 실재 → 이것도 정보다. 표적이 회의가 아닐 가능성(회의는 가장 가시적인 시간 도둑일 뿐) — 재관찰 대상을 비회의 인터럽트(DM·리뷰 대기)로 옮긴다.

## 대안과 기각 사유

- **즉시 처방(전 회의 25분화, 노미팅 수요일):** 기각. 병인이 조각화(B5)면 악화시키고, 편중(B2)이면 무효다. 부활 조건: 1단계 조기 판정 게이트 점화.
- **설문 기반 진단:** 기각. 자기 보고는 불만의 재진술이지 병인 분리가 아니고, 가장 최근의 나쁜 회의 하나가 과대 대표된다. 단 보조로 채택: 회의 종료 1문항("이 회의는 내 참석이 필요했다")을 M5의 교차 검증용으로만.
- **4주 이상 전수 관찰:** 기각. 동결 유지 비용과 측정 부담이 팀의 인지 여유를 초과한다. 소급 4주 + 실측 2주로 같은 시야를 더 싸게 확보. 부활 조건: 가정 2가 깨져 소급 데이터가 무효일 때.

## 리스크 & 완화

- **R1 관찰 효과** — 2주 데이터가 평소보다 착하게 나옴. → 소급 대비 20% 룰로 탐지, 총량 판정은 소급 채택.
- **R2 측정이 새 불만이 됨.** → 부담 상한 + "초과 시 지표를 깎는다" 규칙. 관찰 담당은 1인으로 고정.
- **R3 복수 분기 동시 점화.** → 사전 규칙: 처방은 하나만, 인시 절감 기대량 순. 나머지는 다음 사이클로 이월.
- **R4 동결 기간 중 불만 격화.** → 가정 3의 응급 경로로 전환(팀원 과반 요청 시), 응급 규칙은 동결 대상에 편입.
- **R5 판정 불일치(3단계).** → 재판정 1회 한도. 두 번째도 불일치면 해당 지표는 "판정 불능"으로 기록하고 분기표에서 제외 — 억지 판정보다 정직한 결측이 낫다.

## "완료"의 판정 기준

- **관찰의 완료(실측 문장):** 지표 M1~M6 각각이 분기표의 최소 한 분기를 실제로 갈랐고(고아 지표 0), **두 사람이 데이터만 보고 독립 판정해 같은 분기에 도달한다.**
- **개선의 완료(실측 문장):** 처방 6주 후 재측정에서 점화됐던 분기의 지표가 임계 아래로 내려가고, 간이 설문 "지난주 나의 회의 시간은 적절했다" 동의율이 관찰 종료 시점 대비 20%p 이상 상승한다.
- **부산물:** 미점화·기각된 분기 목록이 "데이터가 기각함"으로 문서화되어, 다음번 같은 불만이 나올 때 논쟁을 처음부터 반복하지 않는다.
