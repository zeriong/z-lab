# 코드 리뷰 프로세스의 제1원리 재설계
- 추론 프레임: first-principles
- 한 줄 요약: 리뷰의 가치를 공리 2개(간극·표본)로 환원하면 리뷰의 산출 단위가 "가부 판정"에서 "두 독립 모델의 간극 보고"로 바뀐다. 병합·수리·배정·생략을 전부 그 단위 위에서 연역한다.

## 프레임이 요구하는 시작점 — 공리, 그리고 공리가 되지 못한 것

이 프레임의 실패 모드는 아는 관행을 공리 언어로 재서술하고 독립 연역이라 착각하는 것이다. 방어를 미리 건다: (1) 모든 설계 결정은 어느 공리에서 나왔는지 소급 표시된다 — 소급되지 않으면 삭제. (2) 연역이 기존 관례와 재회하면 "부합"으로 태그해 공리의 방증으로만 기록하고, **관례와 갈라지는 연역이 하나도 없으면 이 프레임은 미적용으로 자가 판정**한다.

"왜 리뷰가 가치를 만드는가"의 환원:

- **공리 1 (간극 공리).** 결함은 작성자의 의도 모델과 코드의 실제 의미 사이의 간극에 산다. 작성자는 자신의 모델을 통해 코드를 읽으므로 이 간극을 스스로 관측할 수 없다. 간극의 관측에는 **독립적으로 구축된 두 번째 모델**이 필요하다 — 이것이 리뷰가 만드는 첫 번째 가치의 전부다.
- **공리 2 (표본 공리).** 코드 한 줄의 수명 비용 대부분은 작성 이후의 읽기와 수정에서 발생한다. 리뷰어는 그 미래 독자 모집단에서 추출한 첫 표본이며, 리뷰어의 이해 실패는 미래 비용의 가장 싼 조기 측정치다.
- **기각된 후보 (전파 공리).** "리뷰는 지식을 퍼뜨린다." 참이지만 연역 생산성이 낮아 기각 — 이 후보가 만드는 결정(배정 로테이션, 비전문가 노출)은 전부 공리 2의 표본 논리에서 이미 나온다. 공리는 참/거짓이 아니라 생산하는 결정의 수로 뽑는다. 상한은 3이지만 2개로 마감한다 — 겹치지 않는 2개가 연역을 더 멀리 보낸다.

## 층위 이동 — 산출 단위: 가부 판정 → 간극 보고

공리 1을 끝까지 밀면 리뷰는 "의견을 내는 절차"가 아니라 **대조 실험**이다: 같은 코드에 대해 두 개의 독립 모델(작성자의 의도, 리뷰어의 독해)을 만들고 불일치를 관측한다. 따라서 리뷰의 산출물은 가부가 아니라 **간극 보고**이고, 병합 가부는 간극 보고 위에서 규칙으로 계산되는 결과다. 이하 프로세스는 전부 이 단위에서 연역된다.

**블라인드 재진술 프로토콜 (공리 1 — 대조가 성립하려면 순서가 곧 실험 설계다):**
1. 작성자는 변경마다 1~3문장의 **의도문**을 봉인 제출한다("이 변경 후 시스템은 X 상황에서 Y로 동작한다").
2. 리뷰어는 의도문을 보기 전에 코드만 읽고 **재진술**을 먼저 기록한다.
3. 그 다음 대조한다. 불일치가 곧 발견이다.

**간극 척도 — 숫자 전에 의미부터 (운영 정의):**
- **G3 행동 간극:** 재진술 ≠ 의도문이고, 그 차이가 관측 가능한 동작 차이를 함의. → 기본값: 병합 차단. 해제는 코드 또는 의도문 수정 후 재대조 일치.
- **G2 독해 간극:** 결론은 일치하나 리뷰어가 오독 후 복구했거나 특정 지점에서 모델 구축이 지체됨. → 병합 허용. 단 **수리는 산출물(이름·구조·주석·문서)로만 인정** — 리뷰 스레드의 답변은 수리가 아니다. 미래 독자는 스레드를 읽지 않는다(공리 2).
- **G1 모델 무관 지적:** 어떤 모델 차이도 미래 비용 차이도 연역되지 않는 지적. → 기록 후 자동 기각. 공리로 소급되지 않는 코멘트는 이 프로세스에서 리뷰가 아니다.
- **G0 기계 판정 항목:** 형식·타입·테스트 통과 여부. → 인간 리뷰에서 금지, CI로 이관. (관례와 재회 — 부합으로 기록.)
- 경계 분쟁 기본값: G2/G3이 다투면 G3으로 취급한다.

**접면 리뷰 (공리 1 + 단위 사이 탐색):** 작성자의 모델은 diff를 덮는다. 그러므로 간극은 diff 내부보다 **diff와 작성자가 다시 읽지 않은 기존 코드의 접면**(호출부, 깨질 수 있는 불변식, 영속 데이터)에 농축된다. 리뷰 범위 = diff + 접면 목록. 접면 초안은 도구(참조 검색)가 생성하고 작성자가 보정한다. 이것은 diff 중심 리뷰 도구의 관례와 **갈라지는** 연역이며, 시작점에서 걸어둔 자가 판정 조건을 통과시키는 지점이다.

**리뷰 생략 차선 (공리의 대우):** 두 공리의 기제가 모두 점화하지 않는 변경 — 기계 검증이 의미를 완결 보증하고(G0로 닫힘) 미래 인간 독자가 없는 산출물(자동 생성 코드, 락파일) — 에서 인간 리뷰는 순수 지연이다. 생략을 규칙으로 명시한다.

**차선 배정 (고정 비용 대 기대 가치):** 재진술 비용은 변경당 거의 고정이므로, 간극 확률×간극 비용이 큰 변경에만 전체 프로토콜을 건다. 배정은 규칙으로 계산 가능해야 한다: diff가 공개 인터페이스·스키마·영속 데이터·모듈 경계에 닿으면 **A차선**(봉인 의도문 + 블라인드 재진술 + 접면 목록), 그 외 **B차선**(1문장 재진술; 기계 검증 통과 후 24시간 내 재진술이 없으면 병합이 기본값).

**배정 연역 (공리 2):** 표본은 모집단을 닮아야 한다. A차선 리뷰어 2인 중 1인은 도메인 전문가가 아니라 **미래 유지보수자를 닮은 사람**으로 뽑는다 — 전문가의 유창함은 독해 비용을 가린다.

**관례 재부합 기록 + 한계의 정직성:** 변경 단위 소형화(독립 모델 구축 비용은 변경 크기에 초선형으로 증가해 재진술 품질이 급락 — A차선에 크기 상한)와 기계 검증 우선, 두 결론은 좋은 관례와 재회했고 부합으로 기록한다. 순수 연역은 원리적으로 불가능하다 — 작성자의 지식 전체가 선례다. 독립성의 검증은 새로움이 아니라 공리 소급 가능성으로 한다.

## 문제 정의 / 목표

관례의 누적으로 굳은 코드 리뷰 프로세스를, 리뷰가 가치를 만드는 기제 2개에서 다시 연역한다. 목표: 리뷰 산출물이 간극 보고가 되고, 병합·수리·배정·생략이 전부 공리 소급 가능한 규칙으로 계산되는 상태. 대상은 CI 기반이 있는 5~15인 조직 1팀의 파일럿.

## 명시적 가정 목록

- **가정 1 (전체가 걸린 가정): 의도는 1~3문장으로 외재화 가능하고, 의도문과 재진술의 일치/불일치를 제3자가 판별할 수 있다.** 틀리면: 간극 보고가 척도 없는 취향 전쟁으로 퇴행하고 층위 이동 전체가 무효. 1단계의 독립 분류 일치율 실측이 이 가정의 시험대다.
- 가정 2: 테스트·타입·린트 CI가 이미 있다. 틀리면: G0가 인간 리뷰로 역류해 재진술 시간을 잠식 — 이 재설계는 CI 이후의 프로세스다. 없으면 보류하고 CI부터.
- 가정 3: 리뷰어 풀 ≥ 2이고 비전문가 배정 여유가 있다. 틀리면: 공리 2의 배정 연역을 폐기하고 공리 1 연역만으로 축소 운영.
- 가정 4: A차선 프로토콜 비용이 현행 리뷰의 1.5배 이내. 틀리면: 운영 부담으로 사문화. 파일럿에서 실측하고 초과 시 A차선 기준을 좁힌다.

## 접근 및 단계

1. **척도 합의:** G0~G3 운영 정의와 차선 규칙 확정. → 검증: 과거 리뷰 코멘트 30건을 두 명이 독립 분류, 일치율 80% 미만이면 척도 문구를 고쳐 재시도(척도 성립 전 다음 단계 진입 금지).
2. **G0 이관:** 현행 인간 코멘트 중 기계 판정 가능 항목을 CI로 이전. → 검증: 파일럿 기간 인간 코멘트의 G0 비율이 주차별 하락, 최종 10% 미만.
3. **파일럿 (1팀 · 6주 · A차선만):** 봉인 의도문 + 블라인드 재진술 + 접면 목록 가동. → 검증: 건당 소요 시간 실측(가정 4), G3 검출 건수 기록.
4. **게이트 가동:** G3 차단, G2 산출물 수리(후속 커밋 허용, 5영업일 기한), G1 자동 기각, B차선 24시간 기본값. → 검증: 병합 리드타임 분포(중앙값·90분위) 전후 비교, G2 수리의 산출물 착지율.
5. **검출력 점검 + 재귀 적용:** 분기 1회 합성 G3 결함 1건을 주입해 대조가 실제로 잡는지 확인 — 불일치 0이 지속되면 성공이 아니라 검출력 상실을 먼저 의심한다. 그리고 프로세스 문서 자체에 공리 2를 재귀 적용한다: 이 문서의 미래 독자는 신규 합류자다. → 검증: 파일럿 미참여자 1인이 문서만 읽고 변경 5건의 차선·G 분류를 재현, 기존 판정과 대조.

## 대안과 기각 사유

- **페어 프로그래밍으로 리뷰 대체:** 기각. 공리 1이 요구하는 것은 "독립적으로 구축된" 두 번째 모델인데, 페어는 두 모델이 실시간 동기화되어 독립성이 소멸한다 — 간극 관측 기제가 죽는다. 공리 2의 표본도 오염된다(페어는 이미 맥락을 흡수한 독자다). 부활 조건: 간극이 코드가 아니라 문제 정의에 있는 탐색 단계 작업 — 거기서는 페어가 우월하며 별도 차선으로 병존 가능.
- **LLM 리뷰로 인간 리뷰 전면 대체:** 정직한 인정부터 — 공리 1은 "독립 모델"을 요구할 뿐 인간을 요구하지 않는다. 따라서 LLM 재진술을 G3 후보 탐지 전처리로 파이프라인 앞단에 배치한다(G0 이관의 연장). 전면 대체는 기각: 공리 2의 표본은 미래 독자 모집단에서 추출해야 하고, 그 모집단은 인간이다. 부활 조건: 해당 코드의 유지보수 주체가 사실상 도구로 넘어가면 공리 2의 "독자"가 바뀌므로 전체 재연역.

## 리스크 & 완화

- **R1 의례화:** 재진술이 diff 요약 복붙으로 퇴화. → 봉인 순서 강제(의도문은 재진술 제출 후 공개) + 5단계의 합성 결함 주입으로 검출력을 주기 점검.
- **R2 경계 분쟁:** G2/G3 판정 다툼. → 기본값 상향(다투면 G3). 다툼이 잦으면 개인의 문제가 아니라 척도 문구의 결함으로 보고 1단계로 회귀.
- **R3 운영 부담:** 봉인·재진술·접면이 리뷰 기피를 유발. → A차선 한정 + 가정 4 실측 연동, 초과 시 기준 축소. 규칙이 실행자를 짓누르면 구조가 아니라 짐이다.
- **R4 접면 누락:** 틀린 접면 목록은 재진술을 오도한다. → 접면 초안은 도구가 생성하고, 탈출 결함 사후 분석에 "접면 누락" 범주를 상설한다.

## "완료"의 판정 기준

- **실측 문장 1:** 파일럿 6주 종료 시 무작위 표본 A차선 변경 10건 전부에서, 봉인 의도문과 블라인드 재진술이 기록으로 존재하고 G3 간극 0건 상태로 병합되었음이 제3자 감사로 확인된다.
- **실측 문장 2:** 주입한 합성 G3 결함이 재진술 대조에서 검출된다 — 검출 실패 시 이 프로세스는 미완료다.
- G2 수리 착지율: 무작위 G2 지적 10건 중 8건 이상이 스레드가 아니라 코드/문서 diff로 수리되어 있다.
- 독립 분류 일치율 80% 이상 달성 기록이 있다(가정 1의 실측).
- 병합 리드타임 중앙값이 종전 대비 +20% 이내다(운영 부담 상한, 가정 4의 실측).
- 인간 코멘트 중 공리로 소급되지 않는 것(G 분류 불능 지적)이 10% 미만이다.
