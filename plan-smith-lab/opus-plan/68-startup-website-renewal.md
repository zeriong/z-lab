# 스타트업 웹사이트 리뉴얼: 페이지 절반으로
- 추론 프레임: delete-first
- 한 줄 요약: 페이지를 미학으로 다듬지 않고 "방문자가 해야 할 단 하나의 행동"에 기여하는지로 처형 재판에 세워, 구조 레버로 한꺼번에 삭제하고 남은 것만 남긴다.

## 프레임이 요구하는 시작점 — 후보 전부 깔고 유죄 추정으로 재판
리뉴얼이라는 단어를 보면 내 반사 습관은 "무엇을 더 넣을까, 어떻게 더 예쁘게 만들까"다. 이 프레임은 정반대를 요구한다 — 모든 페이지는 유죄로 추정되며, "이 페이지가 없으면 방문자가 단 하나의 행동을 덜 하게 되는가"를 스스로 입증하지 못하면 삭제된다.

**단 하나의 행동:** 방문자가 "첫 대화(데모/도입 상담)를 신청한다". 이 한 문장이 재판관이다.

현재 페이지 후보(전수, 16개): ①홈(랜딩) ②기능(Features) ③솔루션(산업별 다중) ④요금 ⑤고객사례(케이스별 다중) ⑥회사소개/팀(About) ⑦블로그 ⑧리소스/백서 ⑨채용 ⑩문의(Contact) ⑪FAQ ⑫문서(Docs) ⑬통합(Integrations) ⑭보안/신뢰 ⑮변경로그/로드맵 ⑯개인정보처리방침·이용약관(법적).

## 문제 정의 / 목표
페이지가 늘면서 방문자의 시선과 클릭이 분산되고, 운영자는 SEO·업데이트·번역 비용을 페이지 수만큼 계속 지불한다. 목표는 "예쁜 사이트"가 아니라 **행동 집중** — standalone 마케팅 페이지를 16 → 4로 감축하되(지시된 '절반'보다 깊게 — 레버를 끝까지 밀면 마케팅 표면은 랜딩·요금·고객사례 허브·데모 신청 4개뿐), 데모 신청 전환은 떨어뜨리지 않는다.

## 삭제 레버 — 한 장씩 깎지 않고 구조로 한꺼번에 지운다
개별 페이지를 하나씩 저울질하면 정치적 논쟁만 커진다. 여러 페이지를 동시에 불필요하게 만드는 구조 결정 두 개를 먼저 내린다.

- **레버 1 (저장 단위 전환):** "주제당 독립 페이지" → "하나의 롱폼 랜딩의 섹션". 이 한 번의 결정이 기능·FAQ·통합·보안요약·About·후기를 동시에 standalone에서 제거한다. 방문자는 스크롤 한 번으로 반론이 해소되고 CTA로 수렴한다.
- **레버 2 (표면 분리):** "마케팅 표면"과 "제품 표면(로그인 뒤·문서·변경로그)"을 가른다. 이 결정이 문서·변경로그·로드맵을 마케팅 내비에서 한꺼번에 걷어낸다(URL은 docs 서브도메인으로 생존).

## 처형 재판 — 각 페이지 × 대체재 × 부활 조건
| 페이지 | 판결 | 대체재(삭제≠소멸) | 부활 조건 |
|---|---|---|---|
| 홈(랜딩) | 유지 | — | — |
| 기능 | 랜딩 섹션으로 병합 | 랜딩 기능 섹션 + 90초 데모 영상 | 기능군이 3개 이상 완전 분기될 때 |
| 솔루션(다중) | 삭제 | 랜딩 유즈케이스 문단 1개 | 검증된 버티컬 2개 이상 + 세그먼트 유료 트래픽 |
| 요금 | 유지(최소) | — | (역병합) 상담형 요금이면 랜딩 섹션으로 |
| 고객사례 | 허브 1개로 병합 | 랜딩 인라인 인용 2개 + 사례 허브 1페이지 | 개별 심층 사례가 딜 클로징에 반복 인용될 때 |
| About/팀 | 랜딩 섹션으로 병합 | 랜딩 "누가 만드나" 문단 | 투자사·팀 규모가 신뢰 동인이 될 때 |
| 블로그 | 삭제 | 창업자 외부 채널(링크드인 등) | 오가닉이 획득 채널로 실측 검증될 때 |
| 리소스/백서 | 삭제 | 데모 뒤 자료 전달 | 게이티드 리드젠이 전환 채널로 검증될 때 |
| 채용 | 삭제 | 외부 ATS 링크(푸터 1줄) | 상시 채용 5건 이상 |
| 문의 | 삭제 | 데모 신청 CTA + 푸터 이메일 | (없음 — CTA가 문의를 대체) |
| FAQ | 랜딩 섹션으로 병합 | 랜딩 반론처리 섹션 + 데모 대화 | 도입 전 반복 질문이 20개 이상 |
| 문서 | 서브도메인 이동 | docs.* (내비에서 제거) | 구매자가 개발자로 바뀔 때(전제 변경) |
| 통합 | 랜딩 로고스트립으로 병합 | 랜딩 통합 로고 띠 | 통합이 도입 전 상위 질문이 될 때 |
| 보안/신뢰 | 삭제(요약만 병합) | 랜딩 신뢰 요약 + 데모 뒤 보안 문서 | 첫 엔터프라이즈 딜/컴플라이언스 요구 시 |
| 변경로그·로드맵 | 제품 표면 이동 | 로그인 뒤 화면 | (제품 표면 유지) |
| 법적(개인정보·약관) | 유지 | — | (재판 예외 — 법적 바닥이지 마케팅 페이지 아님) |

**생존하는 standalone 마케팅 페이지 4개:** 랜딩, 요금, 고객사례 허브, 데모 신청 페이지. 이 4개가 "단 하나의 행동" 재판을 통과한 마케팅 표면 전부다(카운트 기준은 오직 이 집합).
**마케팅 페이지 카운트 밖(표면 이동·재판 예외):** 문서(docs 서브도메인)·변경로그·로드맵·로그인/앱 진입은 제품 표면으로 이동해 마케팅 내비에서 빠지므로 마케팅 페이지 수에 세지 않는다. 개인정보처리방침·이용약관은 법적 바닥으로 "단 하나의 행동" 재판의 관할 밖이라, 페이지로 남되 마케팅 카운트에는 넣지 않는다(정직한 예외).

## 명시적 가정 목록 (틀렸을 경우의 영향)
- **가정 A (전체가 걸림): 타깃 방문자는 아웃바운드·추천·특정 검색으로 유입돼 "첫 대화를 할지"를 저울질하는 구매 결정자다 — 오가닉 검색 리서처도, 셀프서브 개발자도 아니다.** 틀리면 블로그·솔루션·문서 삭제가 유입 채널 자체를 파괴해 리뉴얼이 트래픽을 죽인다. 삭제 전부가 이 전제 위에 서 있다.
- 가정 B: 셀프서브 결제가 없고 요금은 상담형이다. 틀리면 단 하나의 행동이 "회원가입"으로 바뀌고, 요금 페이지가 전환 핵심으로 부활한다.
- 가정 C: 오가닉/콘텐츠는 아직 검증된 획득 채널이 아니다. 틀리면 블로그·리소스 삭제가 파이프라인을 끊는다.
- 가정 D: 현재 트래픽이 작아 페이지별 정량 A/B 표본이 부족하다. 틀리면(트래픽 충분) 검증을 정성(통화 녹취)에서 정량 실험으로 바꾼다.

## 접근 및 단계 (각 단계 검증 방법)
1. 후보 전수 목록 + 애널리틱스 실측(페이지별 데모 신청 기여, last-touch + assisted). → 검증: 삭제 후보마다 기여 데이터 1개 이상 확보.
2. 처형 재판 실행 — "이 페이지 없으면 데모 신청이 주는가"를 통화 녹취·애널리틱스로 심문. → 검증: 삭제/병합 후보 각각에 대체재 1줄·부활 조건 1줄 기재(위 표 완성).
3. 삭제 레버 적용 — 랜딩 롱폼 구성, 표면 분리. → 검증: 새 사이트맵의 standalone 마케팅 페이지 4개 이하.
4. 리다이렉트 매핑 — 삭제되는 모든 URL을 301로 대체 섹션 앵커/데모 페이지에 연결. → 검증: 구버전 URL 전수 크롤 결과 404 0건.
5. 파일럿(조직이 수용 가능한 방식) — 신버전을 트래픽 50%에 2주 병행 노출, 데모 신청률 비교. → 검증: 신버전 전환율이 구버전 대비 비열등(하락 아님).
6. 확정 또는 롤백 — 하락 시 어떤 삭제가 원인인지 되돌리는 순서를 사전 지정(고객사례 → 요금 → 기능 섹션 순).

## 대안과 기각 사유 (부활 조건)
- **전면 비주얼 리디자인부터:** 기각 — 문제는 미학이 아니라 페이지 과잉으로 인한 행동 분산이다. 부활: 절반 감축 후에도 전환이 낮고 정성 피드백이 "신뢰가 안 감"으로 수렴할 때.
- **페이지 유지 + 내비게이션만 정리(숨김):** 기각 — 숨겨도 페이지는 살아 SEO·업데이트 비용을 먹고 검색·직접링크로 여전히 유입돼 행동을 분산시킨다. "숨김은 삭제가 아니다". 부활: 법무·영업 반대로 삭제가 조직적으로 불가능할 때의 임시 타협.
- **한 장짜리 원페이지로 초압축:** 기각 — 요금·사례·법적 페이지까지 한 장에 밀어넣으면 영업이 인용할 앵커가 사라지고 법적 고지가 섞인다. 마케팅 표면 4개(랜딩·요금·사례·데모)가 적정선. 부활: 트래픽 극소·제품 단순 시.

## 리스크 & 완화
- R1 SEO 붕괴: 삭제 페이지의 검색 유입 소실. → 301 전수 매핑 + 상위 유입 키워드 페이지는 삭제 전 예외 심사(가정 A 재확인).
- R2 세일즈 반발("사례·보안 페이지 없으면 딜 못 닫는다"): → 삭제가 아니라 "데모 뒤 자료/허브"로 이동, 대체재 명시.
- R3 잘못된 단 하나의 행동(가정 B 붕괴): → 재판 전 최근 30일 실제 전환 목표를 CRM·통화로 확정.
- R4 되돌리기 곤란: → 삭제 페이지 콘텐츠·URL 스냅샷 보관, 부활 조건 문서화.

## "완료"의 판정 기준
- standalone 마케팅 페이지가 16 → 4 이하로 사이트맵에서 실측 감소(문서·변경로그·로그인/앱 진입은 제품 표면, 법적 고지는 예외로, 이 카운트에서 제외).
- 구버전 URL 전수를 크롤해 404 0건, 삭제 URL은 100% 301 응답.
- 파일럿 2주간 데모 신청 전환율이 구버전 대비 하락하지 않음(비열등) — "삭제가 행동을 죽이지 않았다"의 실측.
- 삭제·병합된 모든 페이지가 문서에 "대체재 1줄 + 부활 조건 1줄"을 가짐(대체재 없는 삭제 0건).
- 타깃 방문자 전제(가정 A)가 명시적 가정 섹션에 굵게 명시돼 있음.
