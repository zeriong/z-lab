# 신입 개발자 온보딩 "첫 2주" 프로그램
- 추론 프레임: backward
- 한 줄 요약: "2주 후 신입이 혼자 할 수 있어야 하는 것" 5개를 인수조건으로 먼저 못 박고, 커리큘럼과 버디 체계를 그 조건에서만 역산한다.

## 프레임이 요구하는 시작점 — 검증 가능한 인수조건 5개 (앵커)

이 과제를 보자마자 내 반사적 습관은 "1일차 HR·git 세팅, 2일차 아키텍처 강의… 2주차 실습"이라는 **날짜 순 커리큘럼**을 짜는 것이다. backward 프레임은 정확히 그것을 금지한다. 커리큘럼이 먼저 있으면 완료 판정이 "무엇을 배웠나"로 흐려지고, 배움과 능력은 다르다. 그래서 끝 상태(신입이 혼자 할 수 있는 것)를 먼저 고정하고 거기서만 거꾸로 짠다.

**앵커 자격 시험:** 인수조건은 "그 능력이 안 생기면 온보딩을 다시 하거나, 팀 생산성 드래그가 오래 남는" 최대 재작업 조건만 자격이 있다. "알면 좋은 것"은 앵커가 아니다. 각 앵커 옆에 *그것이 죽이는 커리큘럼 대안*을 명시해, 미리 정한 결론으로 back-chaining하는 프레임 실패모드를 차단한다.

- **앵커 1 (마스터 앵커, 전체가 여기 걸림): 혼자 프로덕션에 작은 변경을 배포한다** — 티켓 선택 → 로컬 작업 → PR → 리뷰 통과 → 머지 → 배포 → 배포 후 검증까지. 실패 시 온보딩의 존재 이유(기여자 전환)가 미달 = 최대 재작업. → 죽이는 대안: "2주 읽기 전용 학습/셰도잉만", "신뢰 쌓일 때까지 배포권 보류(2주 read-only 접근)". 이 앵커는 배포권을 **첫 주에** 여는 것을 강제한다.
- 앵커 2: 로컬 개발 환경을 혼자 세우고, 깨지면 혼자 복구한다. 실패 시 매일 버디가 환경을 돌봐야 해 의존이 영구화된다. → 죽이는 대안: "버디가 세팅한 노트북을 넘겨주기(pre-baked)". 넘겨주면 신입은 재구축을 못 배운다 → 신입이 직접 세우고 버디는 지켜본다.
- 앵커 3: 주어진 버그/기능이 *어느 코드·누구 오너*인지 팀원에게 묻지 않고 스스로 찾는다. 실패 시 팀 전체에 상시 방해 세금이 붙는다. → 죽이는 대안: "가이드 코드 투어로 충분", "최신 아키텍처 다이어그램 배포". 정적 지도는 오너십 변화를 못 담는다 → CODEOWNERS·검색·누구에게-묻나 메커니즘을 가르치는 스캐빈저 과제로 대체.
- 앵커 4: 막혔을 때 올바른 채널·형식으로 도움을 요청하고 적시에 에스컬레이션한다. 실패 시 조용한 정체(안 보임) 또는 과도한 질문(방해) 중 하나로 모든 PR이 다시 돌아온다. → 죽이는 대안: "언제든 편하게 물어봐"라는 모호한 규범, "버디를 상시 대기 오라클로". → 명시적 규칙(예: 스스로 30분 시도 후, 정해진 채널에, 시도한 것 포함해 요청)과 하루 2회 오피스아워로 대체.
- 앵커 5: CI 실패/테스트 깨짐을 스스로 진단하는 기본 루프(로그 위치 → 로컬 재현 → 고치거나 증거와 함께 에스컬레이션)를 돈다. 실패 시 모든 red build가 남의 문제가 된다. → 죽이는 대안: "그린 빌드는 인프라팀이 봐준다". → 첫 주에 로그·CI 접근권을 준다.

**역산이 드러낸 구조:** 앵커 1은 독립 능력이 아니라 앵커 2~5 위에 올라앉은 지붕이다. 배포하려면 환경(2)·코드 위치(3)·리뷰 커뮤니케이션(4)·통과하는 CI(5)가 먼저 서야 한다. 이 의존 구조가 커리큘럼 순서를 결정한다.

**오프-앵커 반입 금지:** 5개에 없는 요구(회사 연혁·전체 조직도 암기, 모든 마이크로서비스 이해, 스타일 가이드 정독, 전 사내 툴 셋업)는 참고 자료로 제공할 수는 있어도 **2주 임계경로를 먹거나 평가 마일스톤을 추가할 권리가 없다.** 반입하려면 "이게 어느 앵커의 전제인가"를 먼저 증명해야 한다. 단 하나의 예외: 법적 의무 항목(보안·컴플라이언스 교육 등)은 앵커와 무관하게 비협상 필수이며 별도 트랙으로 실행한다.

## 문제 정의 / 목표

목표는 "2주 동안 무엇을 가르칠까"가 아니라 "2주 끝에 신입이 위 5개를 실측 통과하게 만드는 최소 경로"다. 성공은 커리큘럼 소화가 아니라 앵커 통과다.

## 명시적 가정 목록 (틀렸을 경우의 영향)

- **가정 A (전체가 걸림): 신입이 2주 안에 안전하게 배포할 수 있는, 위험 낮고 롤백 가능한 "작은 변경"과 그런 파이프라인이 실제로 존재한다.** 틀리면 마스터 앵커(1)가 성립 불가 → 역산 전체가 무너진다. 이 경우 앵커 1을 "스테이징/내부 환경 배포"로 정밀화(완화가 아니라 판정 대상의 재정의)해야 한다.
- 가정 B: 버디 1명이 2주간 자기 시간의 약 30%를 낼 수 있다. 틀리면 오피스아워·리뷰 구조가 붕괴해 앵커 4 코칭이 무너진다. 완화: 버디를 2명이 분담.
- 가정 C: 신입은 일반 개발 경험(git·PR·터미널)은 있고 이 조직의 컨텍스트만 없다. 틀리면(완전 주니어) 각 앵커의 사전지식 전제가 깨져 기간·범위를 재산정해야 한다.
- 가정 D: 배포/로그 접근권을 첫 주에 부여하는 것이 보안 정책상 가능하다. 틀리면 앵커 1·5 타임라인이 밀린다. 최소한 "감독 하 배포" 권한이라도 필요.

## 접근 및 단계 (앵커에서 역산한 마일스톤 — 날짜가 아니라 통과가 기준)

1. **끝점 = 앵커 1의 재현.** 마지막 마일스톤은 "제3자가 고른 티켓을 버디 없이 배포, 서로 다른 티켓으로 2회". 검증: 배포 로그·PR 이력으로 2회 독립 배포 확인.
2. **그 직전: 첫 독립 배포 시도.** good-first-issue를 신입이 골라 혼자 진행, 버디는 리뷰어로만. 검증: PR이 버디의 코드 제안 없이 리뷰를 통과.
3. **그 직전: 감독 하 1회 완주.** 버디의 실제 티켓을 페어로 파이프라인 전체를 한 번 통과 — 이때 앵커 2~5가 맥락 속에서 한꺼번에 연습된다. 검증: 신입이 각 단계에서 다음에 뭘 하는지 말로 설명 가능.
4. **기반: 앵커 2~5 개별 착지.** 환경 재구축(2), 코드·오너 스캐빈저(3), 도움요청 규칙 합의(4), CI 실패 재현 훈련(5). 검증: 각 앵커의 실측(아래 "완료" 참조).

**버디 체계(앵커에서 역산):** 버디의 임무는 "친절"이 아니라 앵커별로 정의된다. (a) 환경을 대신 세우지 않는다(앵커 2). (b) 상시 대기 대신 하루 2회 오피스아워로 방해를 배치화한다(앵커 4). (c) 첫 PR들의 리뷰어다. (d) **최종 판정 티켓은 버디가 아닌 제3의 오너가 고르고 리뷰한다** — "독립"을 실측하려면 검증자가 코치와 달라야 하기 때문. 이 분리는 backward 판정에서 역산된 것이다.

## 대안과 기각 사유 (부활 조건 포함)

- 대안 1: 전통적 시간순 커리큘럼(1주 학습·2주 실습). 기각: 앵커에서 역산되지 않고 "가르치기 편한 순서"라 완료 판정이 능력이 아닌 진도가 된다. 부활 조건: 배포 파이프라인이 신입에게 구조적으로 안 열리는 고규제 환경(가정 A·D가 참일 수 없을 때) — 능력 실측이 불가하므로 시간순+체크리스트로 후퇴.
- 대안 2: 3~6개월 롱 온보딩. 기각: 과제가 "첫 2주"로 범위를 못 박았고, 능력 획득을 늦추면 앵커 3·4 실패비용(팀 간섭)이 길게 누적된다. 부활 조건: 도메인상 2주 첫 배포가 본질적으로 불가능한 경우(안전인증 임베디드 등) — 그땐 앵커 자체를 재정의.
- 대안 3: 버디 없는 셀프서브(문서·자동화만). 기각: 앵커 4와 첫 PR 리뷰는 사람 코칭 없이 실측 판정이 안 된다. 부활 조건: 앵커 4를 뺄 수 있는 초소형·비동기 팀.

## 리스크 & 완화

- R1: 앵커 1을 맞추려 "가짜로 쉬운" 티켓을 주고 통과시킴 → 독립성 실측 오염(back-chaining 실패모드 그 자체). 완화: 최종 티켓은 제3자가 선정·리뷰 + 서로 다른 티켓 2회 재현으로 운 배제.
- R2: 오프-앵커 요구(전체 아키텍처 이해 등)가 슬금슬금 필수 커리큘럼에 재진입 → 2주 임계경로 팽창. 완화: 반입 금지 규칙 + "어느 앵커의 전제인가" 증명 요구.
- R3: 개인차로 2주 내 앵커 미달. 완화: 완료는 날짜가 아니라 앵커 통과로 정의 — 미달 시 "무엇을·왜"를 기록해 연장하거나 앵커를 정밀화(비난 아님).
- R4: 버디 번아웃(가정 B). 완화: 오피스아워 배치화, 상시 대기 금지, 부하 분담.

## "완료"의 판정 기준

- 실측 판정문(핵심): **버디가 자리를 비운 상태에서, 신입이 제3자가 고른 real ticket을 골라 PR → 리뷰 통과 → 프로덕션 배포 → 배포 후 검증까지 혼자 완료하고, 2주차에 서로 다른 티켓으로 2회 재현한다.** (앵커 1)
- 앵커 2 실측: 로컬 환경을 의도적으로 삭제 후 문서만 보고 60분 내 재구축.
- 앵커 3 실측: 무작위 버그 리포트 3건에 대해 "어느 파일·누구 오너"를 팀원 질문 없이 정답.
- 앵커 4 실측: 2주간 도움요청이 정해진 채널·형식으로 왔고, 막힘→에스컬레이션 지연이 반나절을 넘지 않음.
- 앵커 5 실측: 일부러 깨뜨린 CI 1건을 로그만 보고 스스로 원인 지목·수정 또는 증거 있는 에스컬레이션.
- 오프-앵커 항목이 문서에 "참고용·임계경로 밖"으로 명시돼 있고, 법적 필수 트랙이 별도로 완료됨.
