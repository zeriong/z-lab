# 엑셀 파워유저의 SQL 전환 8주 학습
- 추론 프레임: misconception-first
- 한 줄 요약: 8주를 SQL 문법 난이도가 아니라 "엑셀 반사가 SQL을 오염시키는 정도(간섭 반경)" 순으로 배치하고, 매주를 예측→충돌→재구성 과제로 오개념을 깨는 교정 일정으로 만든다.

## 프레임이 요구하는 시작점 — 간섭할 엑셀 반사 목록

진입 자백: 학습 플랜을 보자마자 내 반사는 "SELECT→WHERE→JOIN→GROUP BY→서브쿼리→윈도우 함수" 문법 난이도 순으로 커리큘럼을 까는 것이다. 그런데 이게 정확히 misconception-first가 지목한 실패 모드다 — "오개념 목록을 만들어 놓고 결국 난이도 순으로 가르친다." 프레임은 이걸 금지한다. 뼈대는 문법이 아니라, 오개념이 뒤 학습을 오염시키는 순서여야 한다.

먼저 학습자의 기존 반사를 나열한다(교정 대상). 각 항목은 "엑셀 신념 → SQL 현실"의 충돌쌍이다.

- **M1 행 순서 신앙:** 엑셀에서 행 순서는 의미가 있고 고정이다(정렬하면 유지, ROW(), 바로 위 셀 참조). SQL 테이블은 순서 없는 집합이고, ORDER BY 없으면 순서는 미정의.
- **M2 셀=주소 단위:** 엑셀은 A1을 콕 집어 고친다. SQL은 조건(WHERE)에 걸린 집합을 통째로 다룬다. WHERE 없는 UPDATE는 전부를 친다.
- **M3 절차적/헬퍼컬럼 사고:** 채우기-내리기로 한 행씩 누적 계산. SQL은 "무엇을"을 선언한다 — 한 행씩 도는 게 아니다.
- **M4 빈 셀 = 0/"":** 엑셀 빈칸은 합계에서 0, 연결에서 "". SQL NULL은 3값 논리다. `= NULL`은 아무것도 안 걸리고, 산술은 전염되며, 집계는 NULL을 무시한다.
- **M5 VLOOKUP = 조회:** 행마다 첫 일치 하나를 당겨온다(1:1). JOIN은 집합 결합이라 1:다면 행이 불어난다.
- **M6 피벗테이블 = 드래그:** GROUP BY는 집합을 접는 연산. SELECT 컬럼은 그룹키이거나 집계여야 한다.
- **M7 한 장의 넓은 시트:** 모든 걸 한 시트에. 관계형은 키로 나눠 담고 JOIN으로 붙인다.
- **M8 서식=데이터, 매크로 반사:** 날짜는 표시 서식일 뿐 저장 타입이 아니다. VBA 사용자는 커서/루프로 한 행씩 돌리려는 반사가 있다(가장 위험 — 절차 모델을 SQL 안에서 재생산).

교정 순서 원칙: 난이도가 아니라 "이 오개념을 안 고치면 뒤 학습이 얼마나 오염되나(간섭 반경)"로 정렬한다. M1/M2(집합≠셀)가 뿌리 — 나머지가 전부 그 위에 선다. M4(NULL)는 모든 필터·집계를 조용히 망치므로 JOIN보다 앞. M5(JOIN 행 증식)는 다중 JOIN보다 앞.

## 문제 정의 / 목표

엑셀 파워유저가 8주 뒤 "엑셀 뇌로 쓴 SQL"이 아니라 집합·선언 사고로 SQL을 쓰게 만든다. 목표는 문법 암기가 아니라 멘탈모델 교체다.

## 봉인 목록 (지금은 일부러 안 건드림) + 해제 조건

- 윈도우 함수(LAG/LEAD/running total): 봉인. 이유 — 엑셀의 "위 셀 참조/채우기-내리기"로 되돌아가는 뒷문이다. 집합 사고가 서기 전에 열면 학습자가 이걸로 GROUP BY를 영영 회피한다. 해제: M6(GROUP BY 집합 접기)를 스스로 설명하고 셀프조인을 쓸 수 있을 때(7주).
- 커서/프로시저/트리거(절차형 SQL): 봉인. VBA 반사를 강화한다. 해제: 8주 이후, 선언형이 기본이 된 뒤.
- 인덱스·성능 튜닝: 봉인. 질의가 맞기 전 성능은 소음이다. 해제: 정확한 결과를 내는 질의를 쓴 뒤.
- 정규화 이론/DDL 설계, 트랜잭션: 봉인. 해제: M7 이후 필요 시.

## 명시적 가정 (틀렸을 경우의 영향)

- **가정 1(전체가 걸림): 학습자의 엑셀 모델은 "지식"이 아니라 반사다.** 손이 먼저 VLOOKUP/채우기-내리기로 간다. → 틀리면(사실은 초급자, 백지에 가까움) misconception-first는 물릴 게 없어 무력해진다 → 표준 문법 커리큘럼으로 회귀해야 한다. 0주차 진단 과제로 확인.
- 가정 2: 하루 약 1시간, 8주가 현실적. → 틀리면 교정 주기가 늘어지고 충돌→재구성 사이에서 망각이 끼어든다.
- 가정 3: 실제 다중테이블·비깔끔 데이터에 접근 가능(중복키·NULL·1:다 포함). → 틀리면(장난감 단일테이블) 충돌이 안 터진다 — JOIN 행 증식/NULL 쇼크가 안 나 프레임 자체가 공회전한다.
- 가정 4: 표준 방언(Postgres/MySQL/SQLite). → 틀리면 NULL/JOIN 의미가 달라 학습자의 "예측 정답"이 흔들린다.

## 접근 및 단계 (매주 = 오개념 1~2개의 예측→충돌→재구성; 검증법 포함)

- 0주 환경·진단 게이트: 데이터셋에 중복키·NULL·1:다를 미리 심고, 학습자에게 반사 진단 과제를 준다. 검증 — 학습자가 문제를 헬퍼컬럼/VLOOKUP식 언어로 기술하면 가정 1 성립(프레임 적용 가능).
- 1–2주 M1/M2: "이 SELECT의 행 순서를 예측하라" → 3번 실행/업데이트 후 순서가 변함 → 재구성(순서는 테이블 속성이 아님). WHERE 없는 UPDATE 결과 예측. 검증 — 순서를 보장하려면 학습자가 스스로 ORDER BY를 붙인다.
- 2–3주 M3: 누적합을 채우기-내리기로 상상 → 집계/집합 서술로 대체. 검증 — 한 행씩 도는 서술 없이 집합으로 기술한다.
- 3–4주 M4 NULL: `WHERE bonus = NULL` 결과 예측(0행) → IS NULL 재구성. SUM/COUNT에 NULL 섞기 예측. 검증 — `= NULL` 버그를 스스로 진단·수정한다.
- 4–5주 M5 JOIN: 고객 10명·주문테이블 → INNER JOIN 행수 예측 → 47행 충돌 → 1:다 증식 재구성. LEFT vs INNER, VLOOKUP 첫일치 vs JOIN 중복키. 검증 — 아래 판정 문장.
- 5–6주 M6 GROUP BY: 피벗을 드래그로 상상 → 집합 접기로 재구성. SELECT 비집계 컬럼 에러 예측. 검증 — HAVING vs WHERE를 구분한다.
- 6–7주 M7 다중테이블·서브쿼리 + 재충돌(회귀 점검): 2주차 M1/M2를 새 옷 입혀 재출제, 되돌아가는지 확인.
- 7–8주 봉인 해제 + 캡스톤: 윈도우 함수를 "채우기-내리기"가 아니라 "집합 위의 창"으로 재도입. 실무형 과제.

## 대안과 기각 사유 (부활 조건 포함)

- 문법 진행형 커리큘럼: 기각 — 프레임의 실패 모드 그 자체. 엑셀 모델을 남겨둔 채 문법만 얹어 "엑셀 뇌 SQL"을 양산한다(헬퍼컬럼식 서브쿼리, 행별 커서). 부활 — 0주 진단에서 학습자가 백지에 가깝다면(가정 1 붕괴) 이게 오히려 맞다.
- 프로젝트 몰입형(일단 만들며 배우기): 기각 — 충돌이 무작위로 나고 복붙으로 덮인다. 특정 오개념과의 설계된 대면이 없다. 부활 — 7–8주 캡스톤으로, 혹은 자기진단이 강한 학습자에게.
- 엑셀 비유 전면 금지("엑셀 잊어라"): 기각 — 기존 지식이 프레임이 쓰는 연료다(예측이 있어야 충돌한다). 반사는 꺼지지도 않는다. 부활 — 충돌 후에도 오전이(잘못된 전이)가 반복되면 명시적 대조표로 좁힌다.

## 리스크 & 완화

- R1 충돌이 "지적질"로 느껴져 사기 저하 → 예측은 비공개·무채점, "엑셀 반사는 자산, 매핑할 뿐"으로 프레이밍.
- R2 규칙만 외우고 압박 시 회귀 → 간격 재충돌(같은 오개념을 새 변장으로 2주 뒤 재출제).
- R3 봉인 누수(2주차에 윈도우 함수 검색) → 봉인 목록·해제 조건을 학습자에게 명시.
- R4 장난감 데이터로 충돌 불발 → 0주 게이트에서 중복키·NULL·1:다 실측 확인.
- R5 환경 마찰(질의 실행 불가)이 시간을 먹음 → 0주 설치 게이트 통과 전 진도 금지.

## "완료"의 판정 기준

- 실측 판정 문장: **10명 고객처럼 보이지만 중복 주문키 때문에 47행을 내는 JOIN을 실행 전에 주고, 학습자가 "10보다 큰 수"를 예측하고 증식을 설명하면 M5 재구성 완료 — 여전히 10을 예측하면 미완.**
- 신규 2테이블 1:다 데이터 10건에서 INNER/LEFT JOIN 행수를 실행 전 예측, 8/10 정확.
- "주문 없는 고객"을 LEFT JOIN + IS NULL로, 루프/행별 조회 없이 작성.
- 2주차에 교정한 오개념을 7주차 재출제 시 회귀하지 않음.
- 봉인 항목이 문서에 "해제 조건"과 함께 명시돼 있음.

## 프레임 이탈 / 관성 회귀 자백

- 주차 순서가 결국 대략 문법 난이도(WHERE→JOIN→GROUP BY)와 겹친다. 나는 "간섭 반경 순"이라 주장했지만, 간섭 반경과 난이도가 상당 부분 우연히 일치하는 구간이 있어 내 정렬이 정말 오개념 주도인지 난이도 주도인지 완전히 증명하지 못한다 — 실패 모드가 부분적으로 스며들었다.
- 상승 서사 습관: 주차가 단조 상승으로 읽힌다. 학습에서 회귀는 정상인데, 회귀 점검(R2, 6–7주 재충돌)으로 막긴 했으나 뼈대 자체는 여전히 우상향이다.
- 수렴 안전빵: 0주 게이트, 간격 재충돌은 무난한 교육학 정석이다. 구조적 답이 필요한 곳에서 안전 기본값을 골랐을 수 있다 — 특히 R2를 "간격 반복"이라는 흔한 처방으로 덮었다.
- 가장 약한 섹션: 봉인 목록의 윈도우 함수 — "뒷문" 논리는 그럴듯하나, 늦게 가르치는 게 관습이라 그 관습을 오개념 논리로 사후 정당화한 혐의가 있다. 리뷰어라면 여기를 때리겠다.
