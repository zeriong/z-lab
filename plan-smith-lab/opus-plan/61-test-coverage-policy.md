# 테스트 커버리지 정책: 게이트 강제 vs 팀 자율
- 추론 프레임: dialectic
- 한 줄 요약: "게이트냐 자율이냐"를 조직 전체 한 답으로 풀지 않고, 실패 비용 비대칭(폭발반경 × 회복불가능성)을 판정 함수로 먼저 고정한 뒤 경로별로 분할 판정한다.

## 프레임이 요구하는 시작점 — 양 진영 최강 논거 + 판정 함수 고정

(진입 의례: 이 과제를 보자마자 내 반사 습관은 "커버리지 목표치는 두되 CI는 막지 말자"는 무난한 무승부였다. 그런데 그게 정확히 dialectic이 금지하는 실패 모드 — "둘 다 일리 있다"는 판정 없는 비김 — 이다. 그래서 어느 쪽도 미리 정하지 않고, 판정 함수부터 못 박는다.)

**강제파(게이트 강제) 최강 논거.** 커버리지는 "안 짠 테스트"를 드러내는 사실상 유일한 자동 신호다. 자율에 맡기면 마감 압박 하에서 테스트가 가장 먼저 희생되고, 측정이 없으면 그 침식이 보이지 않는다. 특히 결제·인증처럼 되돌릴 수 없는 경로의 결함 하나가 절약한 개발 시간 전부를 한 번에 초과한다. 마감 압박을 이기는 건 개인의 선의가 아니라 머지 차단이라는 구조뿐이다.

**자율파(팀 자율) 최강 논거.** 커버리지 숫자는 "실행된 라인"일 뿐 테스트 품질(단언의 존재)이 아니다. 절대 수치 게이트는 굿하트 법칙을 부른다 — 개발자는 검증이 아니라 숫자를 위한 가짜 테스트를 짜고, 스위트 전체에 대한 신뢰가 무너진다. 무엇을 테스트할지는 리스크를 아는 사람이 판단해야 하며, 게이트의 마찰은 저위험 경로에서는 순손실이다.

**판정 함수(사전 고정).** 두 진영은 서로 다른 실패를 두려워한다. 강제파는 "안 짠 테스트로 결함이 샌다"(Type-II)를, 자율파는 "게이트가 마찰·위장을 낳는다"(Type-I)를 두려워한다. 판정을 실패 비용 비대칭으로 고정한다:

- **Type-II 비용 = 폭발반경 × 회복불가능성.**
  - 폭발반경: 1(개발자 본인/사내 도구) … 5(전 프로덕션 사용자/전 테넌트).
  - 회복불가능성: 1(즉시 롤백, 영속 효과 없음) … 5(불가역 — 자금 이동·데이터 삭제·PII 유출·안전).
- **Type-I 비용 = 마찰(머지 지연) + 위장(가짜 테스트 유발).** 변동성 큰 탐색 코드에서 특히 높음.
- **판정 규칙:** 게이트는 Type-II ≫ Type-I(폭발반경·회복불가능성이 둘 다 높음)일 때만 이긴다. 그 밖에는 자율이 이긴다.

이 함수는 질문 자체를 교정한다. "게이트냐 자율이냐"는 잘못된 질문이고, 옳은 질문은 "어떤 경로가 게이트를 받는가"다. 그래서 답은 자동으로 **분할 판정**이 된다(아래 접근 참조).

## 문제 정의 / 목표

한 조직이 테스트 커버리지 정책을 결정해야 한다. 목표는 두 실패 비용의 합(샌 결함의 비용 + 마찰·위장의 비용)의 기대값을 최소화하는 정책이다. 그리고 그 정책은 조직이 실제로 소화할 수 있고, 나중에 수치로 재검토 가능해야 한다 — 취향 논쟁이 아니라.

## 명시적 가정 목록

- **A1 (전체가 걸린 가정): 실패 비용은 경로마다 비대칭이며, 그 비대칭이 이 결정의 옳은 축이다.** 틀렸을 경우의 영향: 규제상 전 경로 균일 커버리지가 의무(의료기기·항공 등)라면 판정 함수의 축 자체가 무의미해진다 → 분할 판정 폐기, 전면 게이트로 회귀.
- A2: 저장소를 폭발반경·회복불가능성이 유의미하게 다른 경로들로 분할할 수 있다. 틀리면(모든 코드가 균일하게 치명적이거나 균일하게 사소) 분할 판정이 단일 답으로 붕괴한다.
- A3: CI에서 diff(변경분) 커버리지와, 일부 경로의 뮤테이션 점수를 감당 가능한 비용으로 측정할 수 있다. 틀리면 게이트 기제 자체가 없으므로 리뷰 기반 강제로 대체해야 한다.
- A4: 개발자는 절대 수치 게이트를 게이밍한다(굿하트). 틀리면(고신뢰 성숙 팀) Type-I 비용이 낮아져 자율·경량 게이트가 더 넓게 유리해진다.

## 접근 및 단계 — 분할 판정의 실행

경로별 판정(판정 함수의 직접 산출물):
- **고비용 경로**(결제, 인증/인가, 데이터 마이그레이션·삭제, PII 처리): 게이트 승 → 변경분 커버리지 하드 게이트 + 주기적 뮤테이션 테스트.
- **저비용 경로**(사내 도구, 프로토타입, 기능플래그 실험, UI 접착 코드): 자율 승 → 게이트 없음, 팀 판단.
- **중간 경로**: 절대 수치 대신 래칫 — 변경 라인의 diff 커버리지만 요구하고 총 커버리지 하락을 금지. Type-II는 중간이나 절대수치의 Type-I(게이밍)가 크므로, 마찰이 거의 0인 diff 래칫이 함수상 최적이다.

단계:
1. 판정 함수·척도 고정. → 검증: 엔지니어 2명이 표본 모듈 5개를 독립 채점해 각 축 ±1점 이내로 일치(평가자 간 신뢰도). 안 맞으면 척도 문장을 다시 쓴다.
2. 저장소를 3개 비용 티어로 분할하고 라벨을 소유권 파일에 커밋. → 검증: 라벨 없는 최상위 모듈 0개.
3. 티어별 기제 적용(고=하드 게이트, 중=래칫, 저=무게이트). → 검증: 고티어 모듈에 미테스트 분기를 넣은 PR은 CI가 차단하고, 저티어에 동일한 PR은 통과함을 실측.
4. 재검토 트리거 계측. → 검증: 대시보드가 티어별 인시던트 수·뮤테이션 점수·게이트 차단 시간을 추적하고 알림이 연동됨.
5. 1~2개 팀에 1분기 파일럿 후 확대. → 검증: 마찰·유출결함 지표를 파일럿 전 베이스라인과 비교.

재검토 트리거(승자에게도 뒤집힘 조건, 수치 고정):
- 저티어 경로가 한 분기에 프로덕션 인시던트 ≥ 2회 → 폭발반경·회복불가능성을 오채점한 것 → 고티어로 승격.
- 고티어에서 라인 커버리지 > 80%인데 뮤테이션 점수 < 60% → 게이트가 가짜 테스트를 만드는 중 → 뮤테이션 기반 게이트로 전환.
- 게이트로 인한 PR 중앙 차단 시간 > 4시간(고티어) 또는 개발자 마찰 설문이 임계 초과 → 해당 티어 게이트 재검토.
- 재분류 주기: 2분기마다, 또는 주요 아키텍처 변경 시 전 경로 재채점.

## 대안과 기각 사유 (+ 부활 조건)

- **전면 하드 게이트(강제파 순수형):** 기각. Type-II가 무시할 만한 저비용 경로에까지 Type-I(마찰·게이밍)를 부과한다. 부활 조건: A1이 뒤집혀 규제상 전 경로 균일 커버리지가 의무가 되면 전면 게이트가 옳다.
- **전면 자율(자율파 순수형):** 기각. 결제·인증 같은 불가역 고비용 경로를 운에 맡긴다 — 거기서 샌 결함 하나가 전체 마찰 절감을 초과한다. 부활 조건: 조직이 아주 작고 고신뢰이며 모든 경로가 가역적(순수 사내 툴 샵)이면 자율이 전면 승.
- **절대 총 커버리지 % 게이트(예: 80%):** 기각. 굿하트 게이밍과 임의 임계선 문제. diff 커버리지가 더 나은 대체재. 부활 조건: diff 커버리지 도구를 쓸 수 없는 레거시 저장소에서 최후 수단으로만.

## 리스크 & 완화

- R1: 티어 오분류(고비용 경로를 저로 라벨) → 치명 결함 유출. 완화: 모호하면 게이트 쪽으로 기본값, 보안·데이터 경로는 규칙상 자동 고티어, 인시던트 재검토 트리거로 교정.
- R2: 고티어에서도 게이밍(단언 없는 라인 커버) → 가짜 안전. 완화: 고티어 뮤테이션 테스트 + 리뷰 스팟체크.
- R3: 티어 분류 자체가 관료제 오버헤드. 완화: 티어는 3개뿐, 모듈 단위 성긴 분류, 연속이 아니라 2분기마다 재채점.
- R4: 정치적 포획(팀이 게이트를 피하려 자기 코드를 저티어로 로비). 완화: 티어 지정을 횡단 그룹이 소유하고, 폭발반경·회복불가능성은 소유팀이 아니라 규칙으로 채점.

## "완료"의 판정 기준

- 모든 최상위 모듈에 비용 티어 라벨이 커밋되어 있음(라벨 없음 0개).
- **실측 문장:** 결제(고티어) 모듈에 미테스트 분기를 추가한 PR을 열면 CI가 차단하고, 사내 대시보드(저티어) 모듈에 같은 종류의 PR을 열면 CI가 통과한다 — 둘 다 관찰됨.
- 표본 모듈 채점의 평가자 간 일치도 ≥ 80%.
- 재검토 대시보드가 세 트리거 지표(인시던트 수·뮤테이션 점수·게이트 차단 시간)를 알림과 함께 가동 중.
- 파일럿 1분기 결과: 고티어 유출결함이 0으로 수렴하면서 저티어 PR 중앙 차단 시간은 임계 이내 유지.
