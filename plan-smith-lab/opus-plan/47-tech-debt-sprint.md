# 4인 × 8주 기술부채 상환 스프린트
- 추론 프레임: budget-allocation
- 한 줄 요약: 32인주를 고정 포트폴리오 예산으로 못 박고 이자율(주당 새는 시간)로 부채를 정렬하되, 균등 배분을 금지하고 큰 부채는 상환이 아니라 정보 구매(스파이크)로 전환한다.

## 프레임이 요구하는 시작점 — 가치 문장 → 고정 예산 → 초과 후보 원장

(진입 관성 자백: 이 과제를 보자마자 내 반사는 "아픈 부채 목록을 만들고 위에서부터 고친다"였다. budget-allocation은 그걸 금지한다 — 예산을 먼저 못 박고, 후보를 예산보다 많이 깔아 절단을 강제하고, 무엇이 "가치"인지 합의부터 하라고 요구한다.)

**(1) 가치 문장 먼저.** 배분 논쟁은 목표 논쟁의 위장이다. "부채를 줄인다"는 측정 불가이므로 가치를 이렇게 고정한다: **이 스프린트의 가치 = 종료 시점에 팀에서 "매주 새는 개발 시간(이자)"의 총합을 최대한 줄이는 것.** 따라서 순위 통화는 ROI = (막아낸 주당 이자) ÷ (상환 원금, 인주)다.
- 이 목표는 유일하지 않다. 대안 가치: (a) 다가오는 로드맵 기능 X의 선행 차단 해제, (b) 사고 발생률 감소. 만약 조직의 진짜 목표가 (a)라면 원장 전체가 재정렬된다(기능 X를 막는 부채만 이자가 사실상 무한대로 취급). 나는 "팀 전반의 만성 지연"이 문제라는 전제에서 이자 총감을 가치로 택했고, 이 선택을 이해관계자와 먼저 합의하는 것을 1단계로 둔다.

**(2) 고정 예산 — 명목 vs 순.** 명목 예산 = 4인 × 8주 = 32인주. 그러나 원장이 놓치는 통화가 있다: 운영세(on-call·코드리뷰·불가피한 프로덕션 대응·회의). 이걸 먼저 떼지 않으면 32인주는 허구다.
- 운영세: 6인주 (주당 팀 전체 ~0.75인 상당의 유지 업무).
- 버퍼: 4인주 (아래 별도 항목).
- **순 상환 예산 = 32 − 6 − 4 = 22인주.** 이 22인주가 실제 포트폴리오다.

**(3) 초과 후보 원장 (예산보다 크게 — 절단을 강제).** 각 부채에 이자(주당 새는 시간, 팀 합계)와 원금(상환 비용, 인주)을 붙인다. 이자 추정은 반드시 "이 시간이 어디서 새는가" 1문장 근거를 동반한다(근거 없으면 원장에서 제외).

| # | 부채 | 이자(주당 시간) | 원금(인주) | ROI | 비고 |
|---|---|---|---|---|---|
| D1 | CI 느림·불안정(flaky) | 8 | 3 | 최상 | 재실행 대기·재run |
| D2 | 수동 배포(회당 반나절) | 5 | 4 | 상 | |
| D3 | 결제 모듈 테스트 부재 | 4 + 사고위험(변동) | 5~12(불확실) | ? | 정보 구매 대상 |
| D4 | 레거시 인증 로직 재작성 | 3 | 8~20(매우 불확실) | ? | 큰 베팅 → 스파이크 |
| D5 | 로컬 개발환경 셋업 반나절 | 2 | 2 | 상 | |
| D6 | 로깅/관측성 부재 | 3 + 사고 지속↑ | 3 | 상 | |
| D7 | 의존성·보안 패치 밀림 | 1(현재) | 4 | 하 | 만기 리스크 있음 |
| D8 | 프론트 상태관리 난장 | 3 | 6 | 중하 | |
| D9 | DB 마이그레이션 수동·위험 | 1.5 + 사고위험 | 3 | 중 | |

확실 항목 원금 합만 3+4+2+3+4+6+3 = 25인주, 여기에 D3·D4까지 전부 상환하면 40인주를 훌쩍 넘는다. **22인주로는 전부 불가능 → 절단이 강제된다.** 이것이 프레임의 요점이다: 후보가 예산을 넘어야 배분이 진짜 결정이 된다.

## 문제 정의 / 목표
4인 개발팀이 8주 동안 신규 기능을 동결하고 누적 기술부채를 상환한다. 목표는 "부채 제거"라는 무한 과업이 아니라, 고정된 22인주(순) 안에서 **주당 새는 시간의 총 절감을 최대화**하고, 남긴 부채는 "의도적으로 남긴 것"으로 명시하는 것.

## 명시적 가정 (틀렸을 경우의 영향)
- **[전체가 걸린 가정] 팀은 이 스프린트 동안 부채에 전념하며 신규 기능 개발은 동결된다.** 틀리면(병행 개발) 순 예산 22인주가 급감하고 포트폴리오 전체가 무너진다 — 이 경우 플랜을 폐기하고 병행 가능한 예산으로 재산정해야 한다.
- 이자 추정(주당 새는 시간)이 ±50% 내 정확하다. 틀리면 ROI 순위가 뒤집혀 상환 순서를 재정렬해야 한다(첫 완료 항목 실측으로 조기 교정).
- 확실 항목의 원금 추정이 ±30% 내다. 틀리면 버퍼가 소진되고 절단 순서가 발동된다.
- 4인의 스킬이 항목에 유연 배치 가능하다(특정인만 D4를 만질 수 있는 병목이 없다). 틀리면 병렬성이 깨져 실질 예산이 준다.

## 접근 및 단계 (각 단계 검증법)
1. **원장·가치 확정 워크숍(0.5인주).** 후보·이자·원금 추정, ROI 정렬, 가치 문장 합의. → 검증: 4인이 각자 독립적으로 "이 스프린트의 성공"을 한 문장 적어 서로 일치하는가. 각 이자 항목에 "어디서 새는가" 근거가 붙었는가.
2. **큰 베팅을 정보 구매로 전환 — 스파이크 먼저(1주차, 각 1인주 상한).** D4(인증)·D3(결제 테스트)는 원금이 미지이므로 상환하지 않고 지식을 산다. 산출물 = 원금·리스크 추정서 + go/no-go. 사전 확정 판정 수치:
   - D4: 원금 ≤5인주 & 이자 ≥3h/주 확인 → 다음 스프린트 백로그로(이번엔 너무 큼). 원금 >10인주 → 재작성 금지, strangler 이음새만 최소 삽입.
   - D3: 사고 위험이 집중된 20% 경로를 특정 → 그 경로 테스트만 0.5~1인주로 상환, 전면 커버리지는 기각.
   → 검증: 각 스파이크가 기한 내 판정 수치를 산출했는가(초과 시 결과는 "판정 불가"로 확정하고 대상에서 제외).
3. **고 ROI 상환(2~7주차).** ROI 순으로 D1 → D2 → D5 → D6 → D9 … 예산 소진까지. → 검증: 각 항목 완료 시 이자를 before/after 실측(예: CI 재실행 대기시간, 배포 소요시간을 실제 계측). "그 작업이 사라졌는가"로 판정.
4. **버퍼·절단 관리(상시).** 버퍼 잔량을 주간 트래킹. → 검증: 0 도달 시 아래 절단 순서를 자동 발동, 중도 재협상 금지.
5. **마감 주(8주차).** 이자 총 절감 실측, 스파이크가 낳은 후속 백로그 정리. → 검증: 착수 항목 중 반제품(신·구 병존) 0건 확인.

### 버퍼 — 별도 항목 (남는 게 아니라 배정)
4인주. 사용 규칙은 셋뿐: (a) 상환 중 드러난 숨은 원금, (b) 우리 상환이 유발한 회귀 수정, (c) 확실 항목의 원금 초과. **신규 스코프 추가에는 절대 쓰지 않는다.** 버퍼가 0이 되면 절단 순서 발동.

### 사전 확정 절단 순서 (지금 정한다 — 중도엔 정치가 된다)
예산 초과 시 아래 순서로 **항목 통째로** 잘라낸다(균등 절삭 금지):
1. D8 프론트 상태관리(원금 최대·ROI 최저) — 첫 절단.
2. D7 의존성 패치 — 단, **부활 트리거**: 보안 만기/규제 기한이 스프린트 내 도래하면 D7은 절단 대상에서 빠지고 최상위로 승격.
3. D9 DB 마이그레이션.

규칙: 자른 항목의 예산을 남은 항목에 "조금씩" 나눠 붙이지 않는다. 임계질량 이하로 반쯤 상환된 부채는 신·구가 병존해 오히려 이자를 늘린다(전부 아니면 안 함).

## 대안과 기각 사유 (부활 조건)
- **대안 A — 균등 배분(각자 싫은 부채 하나씩).** 기각: 프레임의 핵심 실패모드(균등 절삭)를 정면으로 밟아 모든 축이 임계질량 이하로 떨어진다. 부활 조건: 부채들이 완전히 독립적이고 이자가 균일할 때(현실엔 거의 없음).
- **대안 B — 가장 큰 부채(인증 재작성)에 전 예산 집중.** 기각: 원금이 8~20인주로 미지인 상태의 큰 베팅 → 32인주 예산 파산 위험. 부활 조건: D4 스파이크가 원금 ≤5인주로 확정하면.
- **대안 C — pain score 투표로 정렬.** 기각: "무엇이 가치인가" 합의 없이 감정 투표로 목표 논쟁을 회피한다. 부활 조건: 이자를 정량화할 수 없는 부채(사기·심리적 마찰 등)에 한해 보조 지표로만.

## 리스크 & 완화
- **R1 이자 추정이 희망사항.** → 첫 완료 항목(D1)에서 before/after 실측으로 순위를 조기 교정(측정 마일스톤).
- **R2 스파이크가 분석마비로 늘어짐.** → 각 1인주 상한, 초과 시 "판정 불가"로 확정하고 그 부채는 이번 스프린트 제외.
- **R3 절반 상환된 마이그레이션이 신·구 병존으로 이자를 오히려 증가.** → 임계질량 이하 착수 금지 + 균등 절삭 금지 규칙(위).
- **R4 숨은 통화(신뢰·가시성).** 부채 상환은 눈에 안 보여 이해관계자가 "성과 없음"으로 볼 위험. → 매주 절감한 이자를 "시간(시)"으로 환산해 보고(원장이 그대로 보고 지표가 된다).

## "완료"의 판정 기준
- **분모를 먼저 고정한다: 분모는 팀 전체 원장 이자(30.5h/주)가 아니라, 이번에 착수한 상환 항목(D1·D2·D5·D6·D9)이 원장에서 막을 것으로 추정한 이자 합 = 8+5+2+3+1.5 = 19.5h/주다.** 완료 = 이 19.5h/주의 **최소 80%(≥15.6h/주)를 각 항목 before/after 실측(예: CI 재실행 대기·배포 소요시간)의 합으로 실현**. 목표치 80%는 임의값이 아니라 "착수 항목이 추정대로 이자를 막았는가"의 실현율이며, 이자 추정 오차 가정(±50%)을 감안해 추정치의 대부분이 실측으로 확인돼야 완료로 본다. ← 측정 가능 판정 문장.
- 착수한 모든 항목은 100% 완료 — **신·구 병존 상태로 남은 반제품 0건**.
- 큰 베팅 2건(D3·D4)은 상환이 아니라 "정보 구매 산출물(원금·리스크 추정서 + go/no-go)"로 종료, 무계획 착수 0건.
- 버퍼 4인주 사용 내역이 사용 규칙 (a)(b)(c)에 100% 대응(신규 스코프 유입 0).
- 절단이 발생했다면 사전 확정 순서대로만 집행(중도 정치적 재협상 0건).

## 빠뜨린 범주 점검 (커버리지 스윕)
사람(스킬 병목·번아웃), 프로세스(스프린트 중 신규 부채 유입 동결), 커뮤니케이션(이해관계자 보고=R4), 측정(이자 실측), 미래(스파이크 산출물이 다음 스프린트로 이월), 그리고 "의도적으로 안 건드린 부채"(D7·D8 절단, D4 연기)를 명시했다. 남은 약점: 이자 추정의 초기 정확도 — R1으로만 방어한다.
