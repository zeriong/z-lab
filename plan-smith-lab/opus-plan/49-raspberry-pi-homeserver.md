# 라즈베리파이 1대 홈서버 스택
- 추론 프레임: constraint-first
- 한 줄 요약: RAM·SD 수명·전기·소음·가족 눈치를 먼저 벽으로 세워 그 교집합이 비었음을 증명하고, "전부 자가호스팅"과 "Pi의 디스크=SD"라는 전제를 깨서 살아남는 서비스만 큐레이션한다.

## 프레임이 요구하는 시작점 — 제약을 먼저 벽으로 세운다

솔직히 이 과제를 보자마자 내 반사적 습관은 "awesome-selfhosted 리스트에서 12개 골라 docker-compose 한 방"이다. 기능 목록부터 채우는 것. constraint-first는 정확히 그걸 금지한다. 기능이 아니라 제약을 먼저 못 박고, 그 제약이 무엇을 죽이는지부터 본다.

다섯 개의 벽(각각 실측 단위로 고정):
- **C1 (RAM):** Pi 1대, 가용 RAM ~3GB(4GB 모델 기준, OS 제외). 동시 상주 프로세스 총합이 이 안에 들어와야 함.
- **C2 (SD 수명):** microSD는 쓰기 수명이 유한. 쓰기 폭주(DB WAL, 고빈도 메트릭, verbose 로그)는 카드를 수개월에 죽인다. → 쓰기 예산이 곧 설계 변수.
- **C3 (전기):** 24/7 상시. 벽전력 목표 낮게(idle 8W 이하 급). 상시 회전 디스크·발열 큰 부하 금지.
- **C4 (소음):** 거실/생활공간 무소음. 스피닝 HDD·팬 소음 금지 → 패시브 냉각·플래시 저장 선호.
- **C5 (가족 눈치):** 취미 서버 때문에 집 인터넷이 끊기면 안 됨. 재부팅 한 번에 온 집 DNS가 죽으면 가족 인내심은 첫 장애에 붕괴. 물리적으로도 거실에 흉한 깜빡이 박스 금지.

## 모순 판정 — 교집합이 비었음을 먼저 증명

설계 전에 제약들의 교집합이 공집합인지 본다. 순진한 요구 = {풀 홈클라우드(파일·사진·미디어·홈오토메이션·광고차단 DNS·대시보드·메트릭) 전부 자가호스팅, 24/7, 무소음, 저전력, SD 1장 Pi 1대, 가족 불편 제로}.

이 교집합은 **증명 가능하게 비어 있다**:
- 미디어 라이브러리(대용량 저장) → HDD 필요 → C4(무소음)·C3(전력) 위반.
- 풀 메트릭/로그 + DB 쓰기 → C2(SD 수명) 사망.
- 모든 서비스 동시 상주 → C1(RAM) 초과.
- 광고차단 DNS를 집의 유일 리졸버로 → 재부팅 시 C5(가족) 위반.

교집합이 비었으니 프레임의 명령은 명확하다: 요구를 조금씩 깎지 말고(shave 금지) **구조의 전제를 깬다**. 깰 전제 셋:
- 전제 A "홈서버 = 전부 자가호스팅" → 깬다: 큐레이션한다. 일부는 설계상 클라우드에 남긴다(아래 구멍을 가정으로 선언).
- 전제 B "Pi의 디스크 = SD 카드" → 깬다: 상태(state)는 외장 SSD에 두고, SD는 읽기 위주 OS 부팅용으로 강등.
- 전제 C "집의 필수 서비스도 내 취미 박스가 담당" → 깬다: 가족이 체감하는 필수 서비스에는 Pi가 아닌 폴백을 둔다.

## 문제 정의 / 목표

가정용/개인용 홈서버를 위 다섯 벽 안에서 상시 운영 가능하게. 목표는 "많은 기능"이 아니라 **제약 내 생존 + 가족 인내심 유지**. 살아남는 서비스만 남긴다.

## 명시적 가정 목록 (틀렸을 경우의 영향)

- **가정 1 (전체가 걸림): 이 서버는 가정/개인 규모다 — 사용자 한 자릿수, LAN 우선, 낮은 동시성.** → 틀리면(외부 다수 사용자·사업화) C1 RAM 계산, C2 쓰기 예산, C5 관용까지 전부 붕괴. 이 플랜의 모든 수치가 이 전제 위에 서 있다.
- 가정 2: 외장 SSD를 USB로 붙일 수 있다(전제 B의 물리 근거). → 틀리면(SD 전용) C2 해소책이 무너지고 살아남는 서비스 집합이 급감.
- 가정 3 (클라우드 구멍 1 — 정직히 선언): 오프사이트 백업은 클라우드 오브젝트 스토리지(무료/저가 티어)에 의존. "Pi 1대 자가호스팅"에 낸 구멍. → 틀리면 SSD/SD 동시 사망 시 데이터 영구 손실.
- 가정 4 (클라우드 구멍 2 — 정직히 선언): 외부 원격 접속은 클라우드 터널 릴레이(무료 티어) 의존, 인바운드 포트 개방 없음. → 틀리면 원격 접속 불가(단, LAN 접속은 터널 없이도 살아 있게 설계 → 가족 영향 0).

## 접근 및 단계 (제약이 허용한 것만, 각 단계 검증법)

1. 제약 예산을 숫자로 고정: idle 전력, RAM 헤드룸, SD 주간 쓰기 상한, 소음=무소음, 가족 SLA. → 검증: 베이스라인 idle 전력·RAM 실측.
2. 저장 분리(전제 B 실행): OS/부팅은 SD(읽기 위주), 스왑 비활성, 로그는 tmpfs/용량 제한. 상태·데이터는 외장 SSD. → 검증: 24h SD 쓰기량 측정해 예산 이하 확인, state 디렉터리가 SSD인지 확인.
3. 서비스 큐레이션(포트폴리오 상한): RAM/쓰기 예산에 맞는 생존 집합만 선택(예: 광고차단 DNS, 경량 파일 동기화, 홈오토메이션 정도). → 검증: 전부 상주 시 총 RAM 예산 이하, 각 서비스 쓰기 빈도 측정.
4. 가족 필수 폴백(전제 C 실행): 공유기 보조 DNS를 공용 리졸버로 설정 → Pi 재부팅해도 인터넷 유지. → 검증: **Pi 전원을 뽑아도 집 인터넷이 30초 내 계속 리졸브되는지 실측**.
5. 클라우드 구멍(가정 3·4): 터널로 원격 접속(포트 개방 0), SSD 상태를 매일 암호화해 클라우드에 백업. → 검증: 클라우드 백업에서 새 SD+SSD로 복원 드릴 성공.
6. 가족 인내심 운영: 저사용 시간대 무인 업데이트·재부팅, 서비스 자동 기동, 1페이지 런북. → 검증: 새벽 예약 재부팅 후 수동 개입 0으로 전 서비스 복귀.

## 대안과 기각 사유 + 제약이 죽인 포기 목록

제약이 죽인 것들을 명시한다(부활 조건 포함):
- 대용량 미디어 라이브러리 / NAS: 기각. C4(무소음)+C3(전력)+SSD 대용량 비용 위반. **부활 조건:** Pi+HDD를 소음이 무관한 창고/벽장(냉각 확보)에 둘 수 있으면 C4가 풀림.
- 풀 메트릭/관측(고빈도 스크레이프 + 대시보드): 기각. C2(SD 쓰기)+C1(RAM). **부활:** 상태를 SSD로, 스크레이프 주기를 길게 잡으면.
- 온디바이스 사진/영상 ML·실시간 트랜스코딩: 기각. C1(RAM/CPU). **부활:** 클라우드 또는 더 강한 박스로 오프로드.
- 개인 서비스의 진짜 무중단(HA): 기각. "Pi 1대" 전제가 원천 차단. **부활:** 2번째 노드 도입 시.
- (대안 축) 미니PC/중고 서버로 교체: 기각. C3(전력)·C4(소음) 초과. **부활:** 전기·소음 벽이 이사/공간 확보로 완화되면.

정밀화 vs 완화(정직 공개): "가족 불편 제로"는 "가족 필수(인터넷/DNS)는 그레이스풀하게만 저하되고 하드 장애 없음, 개인 서비스는 잠깐 다운 허용"으로 **정밀화**했다(누구를 보호하는지 특정 = 집의 연결성). "무소음"은 완화하지 않고 그대로 유지. SSD 이전은 "SD가 죽으면 안 된다"를 낮춘 게 아니라 저장 매체 전제를 바꾼 것(SD는 읽기 위주로 정밀화).

## 리스크 & 완화

- R1 정전 시 SD 손상(Pi 고질병): → SD 읽기 위주 + 저널링 FS + 상태는 SSD·클라우드 백업. 전원 불안정하면 소형 UPS.
- R2 USB SSD 전력 부족→불안정: → 충분한 PSU/전원 공급, undervoltage 플래그 클리어 확인.
- R3 클라우드 구멍 붕괴(가정 3·4 파기): → 터널은 원격 전용, LAN 접속은 독립 생존; 백업 타깃 2곳으로 이중화.
- R4 패시브 냉각 스로틀링: → 부하 자체가 이미 제약돼 낮음, 히트싱크/케이스, 온도 실측.
- R5 1인 운영자 부재 시 공백: → 자동 재기동 최대화, 사람 개입 필요 지점을 0에 수렴.

## "완료"의 판정 기준 (실측)

- **Pi 전원 코드를 뽑아도 집 인터넷이 30초 내 계속 DNS 리졸브** (C5 재정의 형태 실측).
- 전 서비스 상주 상태에서 7일 SD 쓰기량이 예산 이하 → SD 예상 수명이 목표 연수 초과(C2 실측).
- 생존 서비스 전부 상주 시 정상상태 RAM이 예산(예: 3.0GB) 이하, 일주일 OOM kill 0(C1 실측).
- idle 벽전력이 목표(예: 8W) 이하(C3 실측).
- 복원 드릴: SSD 초기화 후 클라우드 백업에서 개인 서비스가 1런북 패스로 복귀.
- 거실 1m에서 암소음 위로 들리지 않음(C4, 가족이 판정하는 실질 SLA).
- 포기 목록(대용량 미디어·풀 메트릭·온디바이스 ML·진짜 HA)이 문서에 "제약이 죽임 + 부활 조건"으로 명시돼 있음.
