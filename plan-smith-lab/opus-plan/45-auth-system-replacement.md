# 레거시 모놀리스 인증 시스템 교체
- 추론 프레임: risk-matrix
- 한 줄 요약: "무엇이 제일 무서운가"가 아니라 실패 시나리오를 확률×피해 점수로 정렬하고, 되돌릴 수 없음은 점수 안에서만 반영해 위에서부터 구조적으로 막는다.

## 프레임이 요구하는 시작점 — 실패 시나리오를 확률×피해로 정렬

먼저 두 축의 척도를 두 사람이 독립 채점해도 같은 값이 나오도록 조작적으로 정의한다. 척도가 없으면 정렬은 취향 싸움이 된다.

**피해(D) — 손해의 회복 가능성으로 측정**
- D5: 되돌릴 수 없음. 계정 탈취로 인한 자금 이동·개인정보 유출, 법적 신고 의무 발생. 사후 어떤 조치로도 원상복구 불가.
- D4: 전 사용자 강제 재인증. 전원 재로그인 필요, 진행 중 세션 상태(장바구니·작성 중 문서) 소실. 계정 자체는 안전, 시간이 지나면 정상화.
- D3: 일부 사용자 산발적 인증 실패. 재시도·재로그인으로 개별 복구, 지속 분 단위.
- D2: 신규 로그인만 실패, 기존 세션 유지. 롤백 1회로 전원 복구.
- D1: 내부 지표·로그 오류, 사용자 체감 없음.

**확률(P) — 현 설계대로 실행 시 발생 빈도**
- P5: 방어가 없으면 거의 확실(시스템 기본 동작상 발생).
- P4: 흔한 실수 1개(설정 누락 등)로 발생.
- P3: 조건 2개 이상 겹쳐야 발생.
- P2: 드문 조건 + 실수 조합.
- P1: 이론상 가능하나 현실적으로 희박.

**정렬(위에서부터 커버)**

| 시나리오 | 확률 | 피해 | 점수 | 근거 |
|---|---|---|---|---|
| 전 사용자 로그아웃 | P5 | D4 | 20 | 토큰 서명키·세션 저장소가 바뀌면 기존 세션 전량 무효화는 기본 동작 → P5. 전원 재인증 → D4. |
| 마이그레이션 중 계정 탈취 | P3 | D5 | 15 | 취약 창(재해싱 중 이중 검증·평문 노출)은 방어 없으면 열리나, 실제 탈취는 공격자 행동+창 동시 필요 → P3. 되돌릴 수 없음 → D5. |
| 세션 이중화 불일치 | P4 | D3 | 12 | 구·신 병행 중 세션 동기화 지연·경합은 흔한 실수 1개로 발생 → P4. 산발적 인증 실패, 재시도 복구 → D3. |

**거부한 유혹(프레임의 실패 모드 방어).** 탈취(15)를 "되돌릴 수 없으니 1순위로"라고 올리고 싶었다. 거부한다 — 되돌릴 수 없음은 이미 D5로 점수에 들어갔고, 다시 올리면 이중 계산이다. 공포·난이도는 순위를 바꾸지 못한다.

**위상 예외 — 1회만 사용.** 정상 순서는 로그아웃(20) → 탈취(15) → 이중화(12)다. 그러나 대량 로그아웃을 피하는 유일한 완화책이 "구·신 병행 중 세션을 다리 놓는 것", 즉 세션 이중화 브리지 위에 서 있다. 브리지가 불안정하면 1순위 완화 자체가 성립하지 않는다. 따라서 최하위 이중화(12)가 상위 항목의 인프라이므로 위상 예외를 딱 한 번 적용해 맨 앞으로 당긴다. 예외는 여기서 소진 — 이후 어떤 항목도 이 근거로 순서를 바꿀 수 없다.

**중단선(P×D ≥ 12) 및 커버리지 스윕.** 세 시나리오는 모두 선 위다. 축을 빠뜨렸는지 스윕한 결과: brute-force 방어 공백(P3×D4=12)은 사실상 탈취의 한 벡터라 #탈취에 흡수, M2M·관리자 서비스 토큰 만료(P4×D3=12)는 선상 동급으로 별도 유지. 선 아래는 **의도적 미커버(누락이 아닌 결정)** 로 기록: OAuth 연동 끊김(9), MFA 재등록 요구(9), 감사 로그 단절(8, 비규제 가정 하), 로그인 지연 폭증(6).

## 문제 정의 / 목표

레거시 모놀리스에 내장된 인증 시스템(세션·비밀번호·토큰)을 신 시스템으로 교체하되, 목표는 "완성도"가 아니라 상위 실패 시나리오를 점수 순으로 구조적으로 봉쇄하는 것. 커버리지 100%는 목표가 아니다.

## 명시적 가정 목록

- **가정 1(전체가 걸림): 구·신 시스템이 하나의 세션을 공유·검증할 다리(공유 세션 저장소 또는 검증 위임 API)를 놓을 수 있다.** 틀리면 #1 완화와 위상 예외의 전제가 통째로 붕괴 → big-bang 외 선택지가 사라진다.
- 가정 2: 세션 토큰을 서버측에서 검증 가능(구 서명키를 신 시스템에 read-only 탑재 가능). 틀리면(불투명 서드파티 토큰) 기존 세션 유지 불가 → 로그아웃 D4 확정.
- 가정 3: 비밀번호를 lazy re-hash할 수 있다(로그인 시점에 원문 확보해 재해싱). 틀리면 평문 일괄 처리 필요 → 탈취 창의 P 상승.
- 가정 4: 비규제 서비스라 감사 로그 단절이 D2. 틀리면(금융·의료 규제) 단절이 D4로 올라 중단선 위로 진입 → 커버 대상 재정렬 필요.

## 접근 및 단계 (정렬·위상 예외 순서대로, 각 단계 검증법)

0. **세션 브리지 신뢰화(위상 예외 인프라).** 구·신이 공유 세션 저장소 또는 단방향 검증 위임으로 동일 세션을 같은 사용자로 인식. 검증: 스테이징 카나리에서 구 세션 발급 요청이 신 시스템에서 동일 사용자로 식별되는 비율 100%, 불일치 0건 실측.
1. **로그아웃 회피(#1, 20).** 구 서명키를 신 시스템에 read-only 탑재해 기존 세션을 계속 검증, 신규만 신 키로 발급. 전량 무효화 금지. 검증: 배포 직후 기존 유효 세션 표본 1,000건이 재로그인 없이 200 유지.
2. **탈취 창 제거(#2, 15).** 취약 창 자체를 없애는 구조 — 비밀번호는 lazy re-hash(평문 일괄 금지), 검증 경로는 신 시스템 단일 경로, rate-limit을 첫 배포에 포함. 검증: 경로 감사로 "약한 검증 경로가 동시에 열린 시각" 0초 확인 + 레드팀 탈취 시도 성공 0.
3. **M2M 토큰(선상 동급).** 서비스·관리자 토큰 만료 캘린더 사전 확보 후 무중단 회전. 검증: 배치·결제 연동 계정 토큰이 마이그레이션 창에서 만료되지 않음을 목록 대조.

전 과정은 조직이 흡수 가능한 형태로 — 전체 전환 전 5% 카나리 → 지표 정상 확인 → 단계 확대(verify-then-expand).

## 대안과 기각 사유 (부활 조건)

- **Big-bang 컷오버(전 사용자 일괄 이전 + 세션 전량 재발급):** 기각. #1(20)을 P5×D4로 정면 실현해 전원 강제 로그아웃 확정. 부활 조건: 예약 다운타임이 조직적으로 허용되고 동시 접속이 무시할 수준(심야 배치성 서비스)일 때.
- **병행 없는 스냅샷 복제(포크 후 데이터 복제):** 기각. 세션 브리지 대신 스냅샷을 복제 → 복제 지연 창에서 #3 불일치와 #2 탈취 창이 동시 확대. 부활 조건: 세션이 stateless(자체 검증 토큰)라 공유 저장소가 애초에 불필요할 때.

## 리스크 & 완화

- **브리지 단일점 장애:** 브리지 실패 시 구 시스템으로 자동 폴백(신규 전환 일시 중지). 롤백 1문장 = "신 발급 라우팅 플래그 off".
- **lazy re-hash 미완료 사용자 잔존(장기 미접속):** 유예 창 최소화 후 강제 재설정. 단 강제 재설정은 D4로 튀므로 유예 창 길이를 리스크 지표로 감시.
- **중단선 아래 항목의 사후 부상:** 가정 4가 깨지면 감사 로그 단절이 D4 → 선 위. 이 부활 트리거를 리스크로 등재.

## "완료"의 판정 기준

- **신 인증 시스템으로 전환한 직후, 전환 전 발급된 유효 세션 표본 1,000건이 재로그인 없이 24시간 동안 계속 200 응답을 유지한다**(#1 실측).
- 마이그레이션 창 동안 레드팀의 계정 탈취 시도 성공 0건, 로그상 "약한 검증 경로가 동시에 열린 시각" 0초(#2 실측).
- 스테이징 카나리에서 구 세션↔신 시스템 사용자 식별 불일치 0건(#3·인프라 실측).
- 중단선 아래 4개 항목이 문서에 "의도적 미커버 + 부활 트리거"로 명시돼 있다.
