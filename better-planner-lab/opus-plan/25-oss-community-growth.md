# 오픈소스 프로젝트의 커뮤니티 성장 계획
- 추론 프레임: 인센티브 설계 중심
- 한 줄 요약: 기여자·메인테이너·사용자 각각의 인센티브 구조를 먼저 모델링하고, 인센티브가 정렬되지 않는 정책은 아무리 좋아 보여도 기각한다.

## 프레임이 요구하는 시작점 — 세 주체의 인센티브 모델

성장 전술(홍보, 이벤트, 문서)을 짜기 전에, 이 생태계의 세 주체가 **각자 무엇을 얻으려 움직이는지**를 먼저 모델링한다. 인센티브와 어긋나는 정책은 단기적으로 통해도 결국 붕괴한다.

**주체별 인센티브 (얻는 것 / 치르는 비용)**
- **사용자:** 얻는 것 = 문제 해결(동작하는 도구), 신뢰(유지보수될 것). 비용 = 학습·도입·의존 리스크. → 사용자는 "안정성·문서·응답성"에 반응.
- **기여자:** 얻는 것 = 문제 자체 해결(자기 필요), 평판·이력, 배움, 소속감. 비용 = 기여 마찰(빌드 어려움, 리뷰 지연, 거절의 상처). → 기여자는 "낮은 진입 마찰 + 인정 + 빠른 피드백"에 반응.
- **메인테이너:** 얻는 것 = 프로젝트 영향력, 평판, (때로) 후원. 비용 = 번아웃(무한 이슈·리뷰·지원), 감정 노동. → 메인테이너는 "부담 분산·기여 품질·지속가능성"에 반응.

## 문제 정의 / 목표

오픈소스 커뮤니티를 지속 가능하게 성장시킨다. 목표는 "스타 수·기여자 수"가 아니라 **세 주체의 인센티브가 정렬돼 자가 지속되는 구조**. 인센티브가 어긋나면 성장은 메인테이너 번아웃으로 귀결된다.

## 인센티브 정렬로 정책 심사

각 정책 후보를 "누구의 인센티브를 채우고, 누구의 것을 훼손하나"로 심사한다.

- **정책: "good first issue" 라벨 + 기여 가이드.** → 기여자 진입 마찰↓(기여자 인센티브 충족), 메인테이너는 초기 라벨링 비용은 들지만 이후 기여 유입으로 회수. **정렬 O, 채택.**
- **정책: 빠른 첫 리뷰(예: 48시간 내 반응).** → 기여자 인정 욕구·피드백 충족. 단 메인테이너 부담↑. → **부담 분산(공동 메인테이너·리뷰 자동화) 없이는 메인테이너 인센티브 훼손 → 조건부 채택**(리뷰 부담 분산 정책과 묶어서만).
- **정책: 공격적 홍보로 사용자·이슈 급증.** → 사용자 유입은 좋으나, 기여자·부담 분산 없이 이슈만 늘면 **메인테이너 인센티브 정면 훼손(번아웃)**. → **기각(현 단계)**. 기여자 파이프라인이 준비된 뒤에만.
- **정책: 기여자 인정 체계(기여자 목록·릴리스 노트 크레딧·메인테이너 승급 경로).** → 기여자 평판·소속감 충족, 메인테이너 부담 분산(승급으로 일 나눔). **정렬 O, 강력 채택.**
- **정책: 후원/스폰서십.** → 메인테이너 지속가능성(번아웃 완화). 사용자·기여자엔 중립. **정렬 O, 채택.**

## 명시적 가정 목록 (틀렸을 경우의 영향 포함)

- **가정 1:** 메인테이너 번아웃이 OSS 성장의 최대 붕괴 요인이다. → 이 가정이 "성장보다 부담 분산 우선" 판단의 근거. 틀리면 홍보 우선 전략이 정당화될 수도.
- **가정 2:** 기여자는 금전이 아니라 평판·배움·필요로 움직인다. → 틀리면(금전 필요) 인정 체계보다 후원 배분이 핵심.
- **가정 3:** 인센티브 정렬이 안 된 성장은 지속 불가능하다. → 이 가정이 프레임의 축. 틀리면(외부 자금으로 억지 지속) 다른 모델.

## 접근 및 단계 (인센티브 파이프라인 순서)

1. **메인테이너 지속가능성 먼저:** 부담 분산 장치(리뷰 자동화, 이슈 템플릿, 공동 메인테이너)부터. 성장의 그릇을 먼저 키움.
2. **기여자 파이프라인:** good first issue + 기여 가이드 + 인정 체계로 사용자→기여자 전환 경로 구축.
3. **그 다음 사용자 유입:** 기여자·메인테이너 구조가 준비된 뒤에 홍보. 순서를 지켜야 유입이 번아웃이 아니라 성장이 됨.
4. **승급 경로:** 활발한 기여자를 메인테이너로 승급해 부담을 재분산(1번으로 되먹임하는 선순환).

## 대안과 기각 사유

- **홍보·유입 우선(스타 수 성장):** 기각. 기여자·부담 분산 없이 사용자만 늘면 이슈·지원 요구가 메인테이너 인센티브를 훼손해 번아웃→프로젝트 방치. 인센티브 미정렬의 전형.
- **기여 문턱 높이기(품질 관리):** 부분 기각. 메인테이너 부담은 줄지만 기여자 진입 인센티브를 훼손. 문턱 대신 "가이드+자동화"로 품질과 진입을 양립.
- **금전 보상으로 기여 유도:** 조건부 기각. 가정 2가 맞으면 내재 동기를 밀어낼(crowding-out) 위험. 인정 체계가 우선.

## 리스크 & 완화

- **R1: 성장 속도 압박으로 순서(3번을 앞당김)를 어김.** → "그릇(메인테이너/기여자 구조) 먼저"를 원칙으로 고정, 유입은 파이프라인 준비도에 연동.
- **R2: 인정 체계가 형식화돼 효과 없음.** → 크레딧을 가시적·구체적으로(릴리스 노트 실명, 승급 경로 명문화).

## "완료"의 판정 기준

- 모든 채택 정책이 **세 주체 인센티브 정렬로 심사**를 통과했고, 미정렬 정책(무분별 홍보)은 기각됐다.
- **메인테이너 부담 분산 구조**가 성장(유입)보다 먼저 갖춰졌다.
- 사용자→기여자→메인테이너로 이어지는 **승급 선순환**이 설계돼, 성장이 부담이 아니라 자원이 된다.
