# 텍스트 기반 로그라이크의 절차적 생성 시스템
- 추론 프레임: First-principles
- 한 줄 요약: "왜 절차적 생성이 재미를 만드는가"를 공리 3개 이내로 정의하고, 장르 관례를 인용하지 않고 그 공리에서만 시스템을 도출한다.

## 프레임이 요구하는 시작점 — 공리 정의 (관례 인용 금지)

"로그라이크는 원래 던전을 랜덤 생성하니까"는 금지어다. 대신 **왜 절차적 생성이 재미를 만드는가**를 1차 원리(공리)로 환원한다. 다른 로그라이크가 어떻게 하는지는 참조하지 않는다.

**공리 (3개 이내)**
- **A1 — 재미는 "의미 있는 결정"에서 나온다.** 의미 있는 결정 = 결과가 불확실하고, 선택지 간 트레이드오프가 있고, 정보가 불완전한 선택. (결과가 뻔하면 결정이 아니라 작업.)
- **A2 — 반복 가치는 "예측 불가능하되 학습 가능"할 때 생긴다.** 매번 완전히 같으면(암기) 반복이 무의미하고, 완전히 무작위면(운) 학습이 무의미하다. 재미는 이 사이 — 규칙은 배우되 배치는 매번 다름.
- **A3 — 몰입은 "내 선택이 결과를 지배한다는 감각"을 요구한다.** 죽음이 내 판단 탓으로 귀속돼야 다음 판을 배운다. 운에 죽으면 학습도 몰입도 끊긴다.

이 3공리가 절차적 생성 시스템의 유일한 근거다.

## 문제 정의 / 목표

텍스트 로그라이크의 절차적 생성 시스템 설계. 목표는 "던전을 랜덤으로 만든다"가 아니라 **A1~A3을 최대화하는 생성 규칙**. 무엇을 절차 생성하고 무엇을 고정할지가 공리에서 결정된다.

## 공리에서 도출한 시스템

**A1(의미 있는 결정)에서 도출:**
- 절차 생성의 대상은 "장식"이 아니라 **결정을 만드는 요소**여야 한다. 방의 색깔을 랜덤화하는 건 A1에 기여 0. 대신 **자원·위협·보상의 배치**를 생성해, 매 방이 "위험을 감수하고 보상을 취할까"라는 트레이드오프를 만들게.
- → 생성기는 각 노드에 (위험, 보상, 정보) 3요소를 배치하되 셋이 상충하도록. "안전한 길 vs 보상 큰 위험한 길" 같은 결정을 강제.

**A2(예측 불가+학습 가능)에서 도출:**
- 랜덤화하는 층과 고정하는 층을 분리한다. **규칙(적의 행동 패턴, 아이템 효과)은 고정** → 학습 가능. **배치(어디에 무엇이)는 생성** → 예측 불가. 
- → "고정 어휘 + 가변 문장" 구조. 플레이어는 어휘(개별 요소의 규칙)를 배우고, 매 판 새로운 문장(조합)을 읽는다.

**A3(선택이 결과를 지배)에서 도출:**
- 생성이 **회복 불가능한 불운**을 만들면 안 된다. 예: "시작하자마자 이길 수 없는 적" 배치 금지. → 생성기에 **해결 가능성 보장 제약**(모든 생성 결과는 최소 하나의 생존 경로를 가짐)을 건다.
- → 이건 순수 랜덤이 아니라 **제약 만족 생성**(constraint-based). 랜덤 후 "풀 수 있는가" 검증, 실패 시 재생성.

## 명시적 가정 목록 (틀렸을 경우의 영향 포함)

- **가정 1:** 세 공리가 로그라이크 재미를 충분히 설명한다. → 틀리면(예: "수집·성장의 쾌감" 같은 축 누락) 생성 시스템이 재미의 일부만 겨냥. **공리의 완전성이 전체의 전제.**
- **가정 2:** 텍스트 매체에서도 A1~A3이 동일하게 작동. → 틀리면(텍스트는 공간감이 약해 위험/보상 배치의 체감이 다름) 생성 요소를 텍스트에 맞게 재정의.
- **가정 3:** "해결 가능성 보장" 제약이 계산적으로 실현 가능. → 틀리면 A3 도출이 이론에 그침.

## 접근 및 단계 (공리 → 도출 → 검증)

1. **공리 확정:** A1~A3을 설계 헌법으로. 이후 모든 규칙은 "어느 공리에서 나왔나"를 밝혀야 채택.
2. **생성 대상 선별:** A1 기준으로 "결정을 만드는 요소만" 절차 생성, 장식은 고정/무시.
3. **층 분리:** A2 기준으로 규칙(고정) / 배치(가변) 분리.
4. **제약 생성기:** A3 기준으로 해결 가능성 보장 제약을 건 생성기 설계(랜덤+검증 루프).
5. **공리 역검증:** 플레이테스트에서 죽음이 "내 탓"으로 느껴지는지(A3), 판마다 다르되 배울 수 있는지(A2), 매 방이 결정을 주는지(A1) 확인.

## 대안과 기각 사유

- **"다른 로그라이크처럼 던전+몬스터+아이템 랜덤":** 기각. 관례 인용이라 프레임 위반. 게다가 "왜"를 안 물으면 A1에 기여 없는 랜덤화(장식 랜덤)에 노력을 낭비.
- **순수 랜덤 생성(제약 없음):** 기각. A3 위반 — 회복 불가 불운이 학습·몰입을 끊음.
- **완전 고정 맵 여러 개(수동 제작):** 기각. A2 위반 — 암기되면 반복 가치 소멸.

## 리스크 & 완화

- **R1: 공리가 불완전(재미의 다른 축 누락).** → 5단계 역검증에서 "설명 안 되는 재미/재미없음"이 관찰되면 공리를 보강(단, 관례가 아니라 원리로).
- **R2: 제약 생성기가 무거움(재생성 루프).** → 제약을 생성 단계에 내장(생성하면서 해결 가능성 유지)해 사후 검증 비용 축소.

## "완료"의 판정 기준

- 시스템의 **모든 생성 규칙이 A1·A2·A3 중 하나로 추적**된다(관례 "원래 그렇다"가 근거인 규칙은 0개).
- 생성 결과가 **항상 해결 가능**(A3)하고, **판마다 다르되 규칙은 학습 가능**(A2).
- 플레이테스트에서 죽음이 지배적으로 **운이 아니라 판단 탓**으로 귀속됨(A3 실증).
