# 10년 된 jQuery 관리자 페이지의 점진적 React 전환
- 추론 프레임: 최소 diff 보수형
- 한 줄 요약: 모든 단계가 독립 배포·독립 롤백 가능해야 한다. "한 번에 다시 짜자"는 유혹이 드는 지점마다 그 유혹을 적고 기각한다.

## 프레임이 요구하는 시작점 — 불변 규칙 선언

이 플랜의 최상단 규칙은 "어떤 단계도 다른 단계의 완료를 전제하지 않으며, 각 단계는 그날 배포하고 그날 롤백할 수 있어야 한다"이다. 이 규칙을 어기는 어떤 아이디어도 채택 불가. 아래 모든 단계는 이 규칙을 통과한 것만 남았다.

## 문제 정의 / 목표

10년 된 jQuery 기반 관리자 페이지를 React로 옮긴다. 하지만 관리자 페이지는 매일 내부 운영에 쓰이므로 **한 순간도 못 멈춘다**. 목표는 "React로 만든다"가 아니라 "매 배포가 안전하게 되돌릴 수 있는 상태를 유지하며 서서히 옮긴다".

## 유혹 로그 (빅뱅의 유혹과 그 기각)

전환 계획을 짤 때마다 "차라리 새로 짜는 게 빠르다"는 유혹이 반복적으로 올라온다. 프레임의 핵심은 이 유혹을 억누르는 게 아니라 **명시적으로 기록하고 기각 사유를 남기는** 것이다.

- **유혹 A: "새 React 앱을 통째로 만들고 준비되면 스위치한다."** → 기각. 스위치 시점에 전체가 한 번에 바뀌므로 롤백 단위가 "전부 아니면 전무". 불변 규칙(독립 롤백) 정면 위반. 게다가 준비 기간 내내 두 코드베이스를 이중 유지해야 함.
- **유혹 B: "라우팅을 React Router로 한 번에 갈아엎자."** → 기각. 라우팅은 모든 페이지가 공유하는 축이라, 한 번 바꾸면 모든 페이지가 동시에 영향. 페이지 단위 독립 배포가 깨짐.
- **유혹 C: "이 참에 상태관리도 Redux로 통일하자."** → 기각. 전환과 무관한 아키텍처 개선을 끼워넣는 것. 최소 diff 원칙 위반이며 롤백 표면적을 키움.
- **유혹 D: "jQuery 코드가 지저분하니 옮기면서 리팩터링하자."** → 기각. "옮김"과 "개선"을 한 커밋에 섞으면 버그 발생 시 원인 분리가 불가능. 옮김은 동작 보존만.

## 명시적 가정 목록 (틀렸을 경우의 영향 포함)

- **가정 1:** 페이지들이 서버 렌더링된 개별 HTML이며 페이지 간 결합이 약하다. → 틀리면(강한 전역 상태 공유) 페이지 단위 격리가 불가능해져 프레임 자체가 흔들림.
- **가정 2:** React를 기존 페이지의 특정 DOM 노드에 마운트하는 "섬(island)" 방식이 가능하다. → 틀리면 점진 전환의 기본 메커니즘이 사라짐.
- **가정 3:** 각 배포를 페이지/기능 단위로 뒤에서 켜고 끌 수 있는 최소한의 플래그 수단이 있다. → 틀리면 독립 롤백이 배포 롤백(전체)으로 후퇴.

## 접근 및 단계 (각 단계가 독립 배포·롤백 가능)

1. **스트랭글러 무해 삽입:** 기존 jQuery 페이지 하나의 비핵심 위젯 1개 자리에 React를 마운트하는 "섬"을 넣는다. 실패해도 그 위젯만 영향. 롤백 = 그 마운트 제거.
2. **가장 단순·저위험 페이지 1개 전환:** 트래픽 적고 상태 단순한 페이지를 React island로 완전 대체. 기존 jQuery 버전은 플래그로 즉시 복귀 가능하게 남겨둔다.
3. **반복 — 위험도 오름차순:** 페이지를 하나씩, 각각 별도 PR·별도 배포·별도 플래그로 전환. 순서는 위험 낮은 것부터.
4. **공유 컴포넌트 추출은 필요가 3번 반복된 뒤에만:** 성급한 공통화(유혹 C의 변종) 금지. 세 페이지에서 같은 UI가 나온 뒤에야 컴포넌트로 뺀다.
5. **라우팅 전환은 최후:** 개별 페이지가 거의 다 React가 된 뒤, 마지막에 라우팅을 옮긴다(유혹 B를 끝으로 미룸).

## 대안과 기각 사유

- **빅뱅 재작성(유혹 A):** 위 유혹 로그에서 기각. 롤백 단위가 전부/전무.
- **iframe으로 신구 분리:** 부분 채택 가능하나 기각 우세 — 스타일·인증·통신 경계 비용이 island 방식보다 큼. island가 최소 diff에 더 부합.

## 리스크 & 완화

- **R1: 신구 코드가 같은 페이지에서 전역 상태(jQuery 플러그인, window 변수)를 놓고 충돌.** → island 경계에서 상태 격리, 공유 필요 시 명시적 이벤트로만.
- **R2: 전환이 지지부진해 신구 공존이 장기화(반영구적 하이브리드).** → 각 스프린트당 최소 1페이지 전환을 리듬으로 고정, 공존 기간을 관리 대상으로.
- **R3: 플래그가 쌓여 관리 부채화.** → 페이지 전환 안정화 후 해당 플래그 제거를 완료 조건에 포함.

## "완료"의 판정 기준

- 전환 과정의 **모든 배포가 개별적으로 롤백 가능**했음(배포 로그로 증명, 전면 롤백은 0회).
- 각 페이지 전환이 **동작 보존**(전환 전후 동일 입력 → 동일 결과, 리팩터링 혼입 없음).
- 유혹 A~D가 실제로 발생했을 때 기각 사유가 기록됐고, 규칙을 깬 예외가 있었다면 그 이유가 남아 있음.
